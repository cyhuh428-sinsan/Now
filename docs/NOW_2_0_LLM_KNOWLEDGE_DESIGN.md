# NowNote 2.0 LLM 지식화 설계서

기준일: 2026-06-01

## 문서 지위

- 이 문서는 NowNote 2.0 완료 여부를 확인하는 기준 문서다.
- 2.0의 목표는 1.1부터 1.9까지 흡수한 Obsidian/Google Keep 유용 기능을 통합 점검하고, 장기 LLM 지식화 작업을 운영 가능한 시스템으로 완성하는 것이다.
- 2.1의 Web 그룹 공유 조회 메뉴와 2.2의 그룹원 메신저는 이 문서 범위에서 제외한다.

## 2.0 목표

- Obsidian과 Google Keep에서 NowNote에 꼭 필요하고 유용한 기능은 더 이상 가져올 것이 없도록 1차 흡수를 완료한다.
- 단순 요약 버튼이 아니라 서버 작업 큐 기반의 지식화 작업을 제공한다.
- 브라우저를 닫아도 작업은 서버에서 계속 진행되고, 사용자는 나중에 상태와 결과를 확인한다.
- LLM 또는 로컬 분석 결과는 자동 반영하지 않고 사용자 승인 후 메모 구조에 반영한다.
- 암호화 메모는 기본 분석 대상에서 제외한다.

## 포함 범위

- 1.1-1.9 기능 통합 점검 기준 정리
- 서버 분석 작업 타입 확장
- 긴 작업 큐 상태 확인
- 작업 중단, 재시도, 실패 보류
- 작업 결과 로그와 진행률 표시
- 임베딩 색인 성격의 키워드/관계 인덱스
- 유사 메모 후보
- 중복 메모 후보
- 관계 링크 후보
- 태그/속성 후보
- 지식 건강 점검
- 사용자 승인/반려 흐름
- 승인 전 자동 스냅샷
- 승인 후 결과 메모 반영

## 제외 범위

- 2.1 Web 그룹 공유 조회 메뉴
- 2.2 그룹원 사이의 메신저
- 운영자가 직접 Python 명령어로 분석을 실행하는 방식
- LLM 결과의 무승인 자동 편집
- 암호화 메모 원문 분석
- Google 생태계 종속 기능
- Obsidian 전체 복제

## 화면 흐름

1. 사용자는 Web 설정의 분석 작업 영역에서 작업 유형을 선택한다.
2. 선택 메모 요약은 현재 선택한 지식 메모만 서버 큐에 등록한다.
3. 2.0 지식화 점검과 후보 분석은 암호화되지 않은 지식 메모 묶음을 서버 큐에 등록한다.
4. 작업 목록에서 queued, running, done, failed, cancelled 상태를 확인한다.
5. queued 또는 running 작업은 중단할 수 있다.
6. failed, cancelled, done 작업은 다시 대기열에 넣을 수 있다.
7. done 작업은 결과를 미리 보고 승인 또는 반려한다.
8. 승인 시 먼저 복구 스냅샷을 남긴다.
9. 메모에 연결된 작업은 해당 메모에 결과를 추가한다.
10. 전체 지식화 작업은 새 루트 메모로 승인 결과를 남긴다.

## 서버 작업 타입

| 작업 타입 | 목적 |
| --- | --- |
| `memo_summary` | 선택 메모 요약 |
| `tree_note_index` | 계층 메모 색인 |
| `recording_summary` | 녹음 전사 요약 |
| `knowledge_2_0_review` | 2.0 지식화 통합 점검 |
| `embedding_index` | 임베딩/키워드 색인 후보 |
| `similar_notes` | 유사 메모 후보 |
| `duplicate_candidates` | 중복 메모 후보 |
| `relation_suggestions` | 관계 링크 후보 |
| `tag_property_suggestions` | 태그/속성 후보 |
| `knowledge_health` | 지식 건강 점검 |

## 결과 구조

분석 결과는 `analysis_jobs.result_json`에 JSON으로 저장한다.

필수 성격:
- `progress_percent`
- `approval_required`
- `apply_mode`
- `logs`
- `summary`
- `suggestions`

후보 성격:
- `embedding_index`
- `similar_notes`
- `duplicate_candidates`
- `relation_candidates`
- `tag_property_suggestions`
- `knowledge_health`

## 승인 원칙

- LLM/분석 결과는 자동으로 기존 메모를 수정하지 않는다.
- 사용자가 승인해야 메모에 반영한다.
- 승인 직전에 `before-analysis-approval` 스냅샷을 남긴다.
- 반려한 작업은 `cancelled` 상태와 반려 메시지를 남긴다.
- 중단/재시도는 API와 Web 화면에서 처리한다.

## 완료 기준

- Web에서 분석 작업 유형을 선택할 수 있다.
- Web에서 2.0 지식화 점검 작업을 큐에 등록할 수 있다.
- 서버는 2.0 지식화 작업 타입을 허용한다.
- 서버 작업자는 2.0 지식화 결과 JSON을 생성한다.
- 작업 목록에서 진행 상태와 결과 미리보기를 확인할 수 있다.
- 작업을 중단하고 재시도할 수 있다.
- 사용자는 결과를 승인 또는 반려할 수 있다.
- 승인 전 자동 스냅샷이 남는다.
- 암호화 메모는 지식화 묶음에서 제외된다.
- 검증 스크립트가 Web 표면과 서버 API 확장을 확인한다.
- 이 문서에 따라 2.0 완료 여부를 판단할 수 있다.

## Obsidian/Google Keep 흡수 완료 판정

NowNote 2.0 시점에서 추가로 가져올 필요 기능은 없다.

이 판정은 “모든 기능을 복제했다”는 의미가 아니라, NowNote 제품 방향에 필요한 기능은 1.1-2.0에서 흡수했고 나머지는 의도적으로 제외했다는 의미다.

의도적 제외:
- Google 계정, Calendar, Tasks, Docs 직접 연동
- 위치 기반 리마인더
- Obsidian 플러그인 생태계 복제
- LLM 자동 대량 편집
- 암호화 메모 원문 서버 분석

## 후속 버전

### 2.1: Web 그룹 공유 조회

- 내 공유 메모 조회/수정
- 전체 그룹원 공유 메모를 지식체계별로 조회
- 그룹 구성원별 공유 문서 조회
- Web 전용 조회 메뉴로 구현

### 2.2: 그룹원 메신저

목표:
- 같은 그룹 사용자 사이에서 메모 협업과 별도로 짧은 대화를 주고받을 수 있게 한다.
- 공유 메모를 수정하지 않고도 의견, 확인 요청, 작업 상태를 남길 수 있게 한다.

포함 후보:
- 그룹원 목록 기반 1:1 메시지
- 그룹 전체 공지 또는 그룹 채팅
- 공유 메모 링크 첨부
- 읽음/안 읽음 상태
- 알림/미확인 메시지 표시
- 관리자 또는 그룹장 기준 차단/보관/삭제 정책
- 서버 보관 기간과 백업/복구 기준

초기 제외:
- 외부 메신저 연동
- 파일 대용량 전송
- 음성/영상 통화
- 공개 채널형 커뮤니티 기능
