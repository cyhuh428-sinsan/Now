# NowNote 1.1 설치형 PC 클라이언트 설계서

기준일: 2026-05-31
확정일: 2026-05-31

이 문서는 NowNote 1.1 작업의 기준 문서입니다.
NowNote 1.1 완료 여부는 이 문서의 완료 기준과 체크리스트로 판단합니다.

## 문서 지위

- 이 문서는 NowNote 1.1 설치형 PC 클라이언트 작업의 기준 설계서입니다.
- 신산님 확정 기준으로 1.1 작업의 완료 여부를 판단합니다.
- 구현 중 판단이 필요하면 이 문서를 우선 기준으로 삼습니다.
- 이 문서에 없는 큰 구조 변경은 제안만 하고, 확정 전에는 구현하지 않습니다.
- 완료 판정은 `완료 기준`과 `완료 체크리스트`가 모두 충족되는지를 기준으로 합니다.

## 1.1 목표

NowNote 1.1의 핵심 목표는 Windows 설치형 프로그램을 Web 화면 포장물이 아니라 PC 로컬 문서와 서버 공유 문서를 함께 다루는 전용 클라이언트로 분리하는 것입니다.

사용자는 설치형 프로그램에서 PC 안의 원본 문서를 안정적으로 관리하고, 필요한 문서만 서버 공유 상태로 선택합니다.
Web은 서버에 공유된 문서만 다루며, PC 로컬 원본 문서를 직접 보거나 관리하지 않습니다.

## 제품 구분

| 구분 | 1.1 기준 역할 | 다루는 데이터 |
| --- | --- | --- |
| Web | 서버 부속 화면 | 서버에 공유된 내 문서 |
| Windows 설치형 | PC 전용 클라이언트 | PC 로컬 원본 문서 + 서버 공유 문서 |
| 모바일 앱 | 빠른 기록 클라이언트 | 휴대폰 로컬 기록 + 서버 공유 문서 |
| 서버 | 동기화와 운영 시스템 | 공유 문서, 사용자, 기기, 녹음, 분석 작업 |

## 현재 상태

- `desktop/main.cjs`는 Electron 창에서 `desktop/app/index.html`을 엽니다.
- `desktop/scripts/sync-web-assets.mjs`는 `web` 정적 파일을 `desktop/app`으로 복사합니다.
- `desktop/preload.cjs`는 `window.nownoteDesktop`에 플랫폼, Electron 여부, 설치형 로컬 저장소 API를 노출합니다.
- 설치형은 Electron main process의 `nownote-desktop-store.json` 파일 저장소를 우선 사용합니다.
- 기존 브라우저 localStorage 데이터는 설치형 파일 저장소가 비어 있을 때 1회 복사 대상입니다.
- 서버 동기화 로직은 기존 `/api/v1/sync` 흐름을 유지하되, 설치형 자동 동기화 켜기/끄기 설정을 추가했습니다.
- 새로 만든 PC 로컬 지식 메모는 기본적으로 서버 공유 대상에서 제외하고, 이미 서버에 공유된 메모를 공유 해제할 때만 서버 삭제 신호를 보내도록 분리했습니다.
- 로컬과 서버가 모두 바뀐 문서는 자동 덮어쓰기를 막고 설정 화면의 충돌 문서 목록에서 처리하도록 보강했습니다.
- 암호화 메모는 서버 분석 큐 등록 전에 차단해 암호문이나 복호화 평문이 분석 입력으로 넘어가지 않도록 했습니다.
- 삭제 보관함 항목은 로컬 보관 상태로 유지하고 서버 동기화 대기 개수에 포함하지 않는 기준을 확인했습니다.
- Web은 로그인 후 서버 공유 문서를 기준으로 화면을 교체하고, Web 화면에서는 공유 상태 토글을 잠가 공유/비공유 관리를 설치형에 맡기는 기준을 확인했습니다.
- 기존 Web, Markdown 가져오기/내보내기, JSON 백업/복원 흐름은 자동 점검 스크립트로 회귀 여부를 확인했습니다.
- NSIS 설치 파일 산출물, `win-unpacked/NowNote.exe` 실행 smoke, 테스트 전용 폴더 설치 후 실행/제거 점검을 통과했습니다.
- `desktop/README.md`, `web/README.md`, `web/help.html`, `docs/PROJECT_STATUS.md`, `docs/WORK_PROGRESS.md`에 1.1 설치형 저장/공유/동기화 기준을 반영했습니다.
- 설치형 파일 저장 실패는 토스트 경고뿐 아니라 설정 화면의 PC 로컬 저장소 상태에도 실패로 남도록 보강했습니다.
- 테스트 전용 설치형 userData 경로에서 메모 작성, 파일 저장, 앱 재시작 후 재로드를 자동 점검했습니다.
- 브라우저 정책 스모크로 공유 안 함 로컬 메모 서버 제외, 공유 메모 전송, 공유 해제 삭제 신호, 삭제 보관함 서버 제외, 충돌 기록, 암호화 메모 분석 차단을 확인했습니다.

## 기본 원칙

- 기존 Web 디자인과 주요 사용 흐름은 유지합니다.
- Web은 서버 공유 문서 전용 기준을 유지합니다.
- 설치형은 PC 로컬 원본을 우선합니다.
- 사용자가 Python, DB, CLI를 직접 실행하지 않아도 상태를 확인할 수 있어야 합니다.
- 동기화, 충돌, 삭제, 암호화 상태는 화면에서 확인 가능해야 합니다.
- 기존 사용자 데이터는 자동 삭제하지 않습니다.
- 서버 API 대규모 변경 없이 1.1을 시작합니다.

## 사용자 흐름

### 단독 설치형 사용자

1. Windows 설치형 프로그램을 실행합니다.
2. 서버 연결 없이 PC 로컬 문서를 작성합니다.
3. 모든 원본 데이터는 PC 로컬 저장소에 저장됩니다.
4. 사용자는 저장 상태와 저장 위치를 화면에서 확인합니다.
5. JSON 백업/복원과 Markdown 가져오기/내보내기를 계속 사용할 수 있습니다.

### 서버 연결 설치형 사용자

1. Web에서 가입하거나 개인 서버 계정을 준비합니다.
2. Web에서 앱/설치형 연결 토큰을 발급합니다.
3. 설치형 프로그램 설정에 서버 주소, 사용자 ID, 연결 토큰을 입력합니다.
4. 설치형 프로그램은 PC 로컬 문서를 원본으로 유지합니다.
5. 사용자는 메모별 공유 여부를 선택합니다.
6. 공유 대상 문서만 서버와 동기화합니다.
7. 충돌이 생기면 설치형 화면에서 상태와 선택지를 확인합니다.

### Web 사용자

1. 서버 주소에 접속합니다.
2. 사용자 ID와 비밀번호로 로그인합니다.
3. 서버에 공유된 내 문서만 봅니다.
4. PC 로컬 원본, 삭제 보관함 원본, 공유 안 함 문서는 보지 않습니다.

## 기능 요구사항

### 1. 설치형 실행 환경 식별

- Electron 실행 여부를 명확히 판단합니다.
- Web 서버 제공 화면과 설치형 화면의 동작 분기 기준을 고정합니다.
- 설치형에서만 보이는 로컬 저장 상태 UI를 제공합니다.
- Web에서는 설치형 로컬 저장 기능이 노출되지 않아야 합니다.

완료 기준:
- 설치형 실행 시 화면 또는 설정에서 설치형 모드임을 확인할 수 있습니다.
- Web 실행 시 설치형 전용 저장 기능이 보이지 않습니다.
- 검증 스크립트가 두 실행 모드의 핵심 UI 차이를 확인합니다.

### 2. PC 로컬 저장소

- 설치형 전용 로컬 데이터 파일을 Electron main process가 관리합니다.
- renderer는 직접 파일 경로를 다루지 않고 preload API를 통해 저장/불러오기를 요청합니다.
- 저장 파일 위치는 사용자 화면에서 확인할 수 있습니다.
- 저장 실패는 조용히 무시하지 않고 화면 경고로 표시합니다.
- 기존 localStorage 데이터는 1회 마이그레이션 대상으로 유지합니다.

완료 기준:
- 설치형에서 메모를 작성하고 앱을 재시작해도 데이터가 유지됩니다.
- 저장 파일 위치를 화면에서 확인할 수 있습니다.
- 저장 실패를 감지하고 사용자에게 표시합니다.
- 기존 localStorage 데이터가 삭제되지 않고 파일 저장소로 가져와집니다.

### 3. 로컬 문서와 공유 문서 분리

- 설치형은 PC 로컬 문서를 원본으로 관리합니다.
- 지식 메모는 공유/공유 안 함 상태를 가질 수 있습니다.
- 공유 안 함 문서는 서버로 전송하지 않습니다.
- Web은 공유된 문서만 조회/편집합니다.
- 일자별 메모의 서버 공유 정책은 기존 동작을 유지하되 설치형에서 상태를 명확히 표시합니다.

완료 기준:
- 설치형에서 공유 안 함 메모가 서버 동기화 대상에서 제외됩니다.
- 공유 메모는 서버와 동기화됩니다.
- Web에서는 공유 안 함 문서가 보이지 않습니다.
- 공유 상태가 메모별로 화면에 명확히 표시됩니다.

### 4. 자동 동기화

- 설치형에서 자동 동기화 기준을 제공합니다.
- 자동 동기화는 사용자가 끌 수 있어야 합니다.
- 동기화 중, 성공, 실패, 보류 상태를 화면에 표시합니다.
- 마지막 동기화 시각과 동기화 대상 개수를 확인할 수 있어야 합니다.
- 수동 동기화 기능은 유지합니다.

완료 기준:
- 설치형에서 자동 동기화를 켜고 끌 수 있습니다.
- 자동 동기화 실패가 화면에 표시됩니다.
- 수동 동기화와 자동 동기화가 같은 상태 기준을 사용합니다.
- Web의 서버 공유 문서 흐름은 기존대로 동작합니다.

### 5. 충돌 처리

- 서버 문서와 로컬 문서가 모두 바뀐 경우 충돌 상태로 표시합니다.
- 자동 덮어쓰기는 하지 않습니다.
- 사용자는 로컬 유지, 서버 내용 적용, 나중에 결정 중 하나를 선택할 수 있어야 합니다.
- 암호화 메모는 평문 비교 없이 안전한 보류 기준으로 처리합니다.

완료 기준:
- 충돌 문서가 목록 또는 설정 화면에서 확인됩니다.
- 충돌 해결 선택 후 동기화 상태가 정상으로 돌아옵니다.
- 암호화 메모 충돌은 원문 노출 없이 처리됩니다.
- 충돌 처리 중 기존 메모 내용이 임의로 사라지지 않습니다.

### 6. 암호화 메모

- 기존 `NOW_ENCRYPTED_V1` 형식을 유지합니다.
- 암호화 키는 서버, 브라우저 저장소, 설치형 저장소에 저장하지 않습니다.
- 암호화 메모는 기본적으로 서버 분석 대상에서 제외합니다.
- 잠금, 복호화, 암호화 해제 흐름을 기존 방식과 호환되게 유지합니다.

완료 기준:
- 기존 암호화 메모가 설치형에서 열리고 잠금 상태로 표시됩니다.
- 올바른 키 입력 시 복호화됩니다.
- 암호화 해제 시 평문 메모로 저장됩니다.
- 서버 분석 요청에서 암호화 메모가 보호됩니다.

### 7. 삭제 보관함

- 설치형 삭제 보관함은 PC 로컬 복원/영구 삭제 기준을 우선합니다.
- 서버에서 삭제된 문서와 로컬 삭제 보관함 항목을 섞어 표시하지 않습니다.
- 삭제 보관함 항목은 사용자가 복원하거나 영구 삭제할 수 있습니다.
- 삭제 보관함 항목은 의도 없이 서버로 재전송되지 않아야 합니다.

완료 기준:
- 삭제한 지식 메모가 설치형 로컬 삭제 보관함에 보관됩니다.
- 복원 시 원래 계층 또는 가능한 위치로 돌아옵니다.
- 영구 삭제 시 로컬 저장소에서 제거됩니다.
- 삭제 보관함 항목이 서버 동기화 대기 개수에 포함되지 않습니다.

### 8. 설치 품질

- NSIS `.exe` 설치 파일 품질을 먼저 안정화합니다.
- 설치 후 시작 메뉴와 바탕화면 바로가기가 정상 동작해야 합니다.
- 설치형 앱 아이콘이 기본 Electron 아이콘으로 보이지 않아야 합니다.
- `.msi`와 자동 업데이트는 1.1 후속 후보로 둡니다.

완료 기준:
- `desktop/dist/NowNote-Setup-0.1.0-x64.exe` 생성이 통과합니다.
- 설치 후 앱 실행이 통과합니다.
- 설치형에서 로컬 저장소와 서버 연결 화면이 정상 표시됩니다.
- `.msi`와 자동 업데이트는 문서상 후속 항목으로 남아 있습니다.

## 기존 기능 유지 기준

- Web 로그인, 직접 가입, 비밀번호 재설정, 연결 토큰 발급 흐름은 유지합니다.

- 설치형의 기존 localStorage 사용자 데이터는 삭제하지 않습니다.
- JSON 백업/복원과 Markdown 가져오기/내보내기는 유지합니다.
- 메모 단위 암호화 형식 `NOW_ENCRYPTED_V1`은 변경하지 않습니다.
- 서버 `/api/v1/sync` 요청/응답 형식은 1.1 초반에는 변경하지 않습니다.
- 모바일 앱의 기존 서버 동기화, 음성 메모, 암호화 메모 읽기 흐름은 깨뜨리지 않습니다.

## 의심되는 부분

- 기존 `web/app.js`가 Web과 설치형을 모두 담당해 분기 조건이 더 복잡해질 수 있습니다.
- localStorage에서 파일 기반 저장소로 옮길 때 중복 저장과 마이그레이션 타이밍 문제가 생길 수 있습니다.
- 설치형 자동 동기화가 기존 수동 동기화 흐름과 충돌할 수 있습니다.
- 삭제 보관함의 로컬 삭제와 서버 삭제 표시가 같은 UI에서 혼동될 수 있습니다.
- 암호화 메모 충돌 처리에서 평문을 보지 않고 비교해야 하므로 병합 기준이 제한됩니다.
- 설치형 저장소 장애가 생기면 사용자가 파일 위치와 오류 원인을 화면에서 알아야 합니다.

## 구현 순서

1. Electron preload에 설치형 저장소 API를 추가합니다.
2. Electron main process에 로컬 JSON 저장 파일 읽기/쓰기 IPC를 추가합니다.
3. 설치형 실행 시 renderer가 localStorage보다 설치형 저장소를 우선 사용하게 합니다.
4. 기존 localStorage 데이터를 설치형 저장소로 1회 가져오는 마이그레이션을 추가합니다.
5. 화면 설정에 설치형 로컬 저장 상태와 저장 파일 위치 확인 영역을 추가합니다.
6. 공유 상태와 동기화 상태 표시를 설치형 기준으로 보강합니다.
7. 충돌 상태 표시와 해결 흐름을 추가합니다.
8. 삭제 보관함과 암호화 메모의 동기화 보호 기준을 검증합니다.
9. 설치형 README와 사용자 도움말을 갱신합니다.
10. 설치 파일을 재생성하고 실행 점검합니다.
