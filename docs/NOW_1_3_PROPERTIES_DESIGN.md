# NowNote 1.3 속성 기반 지식 관리 설계서

기준일: 2026-06-01

이 문서는 NowNote 1.3 완료 여부를 판단하는 기준 문서입니다.

## 목표

메모를 단순 글이 아니라 상태, 중요도, 유형, 출처, 작성자, 마감일, 프로젝트를 가진 지식 항목으로 관리합니다.
Obsidian Bases/Properties의 장점을 NowNote 화면 흐름에 맞게 흡수하되, 서버 인덱스나 별도 데이터베이스 구조는 만들지 않습니다.

## 포함 범위

- 지식 메모별 속성 편집 패널
- 속성 기반 목록 보기
- 상태/중요도/검색 필터
- 상태, 중요도, 유형, 프로젝트, 공유 상태 기준 그룹화
- 속성 필터 저장과 다시 열기
- 누락 속성 점검
- 속성이 포함된 템플릿 메모 생성 1차
- Web과 설치형에서 같은 `node.properties` 구조 사용

## 제외 범위

- 서버 측 속성 인덱스
- 수식, 계산 필드, 관계형 테이블
- 모바일 속성 화면 동등화
- Google Keep의 핀, 색상, 리마인더
- Canvas, Publish, 장기 임베딩

## 데이터 기준

각 지식 메모는 다음 속성을 가질 수 있습니다.

- `status`: idea, active, waiting, done
- `priority`: low, normal, high
- `type`
- `source`
- `author`
- `dueDate`
- `project`

기존 메모는 열릴 때 빈 속성으로 정규화합니다.
속성을 수정하면 해당 메모는 기존 편집 흐름과 동일하게 `pending` 동기화 상태가 됩니다.

## 화면 기준

- 메모 편집 화면에서 속성을 바로 수정할 수 있어야 합니다.
- 속성 보기 팝업에서 여러 메모를 속성 기준으로 비교할 수 있어야 합니다.
- 필터와 그룹을 저장해 반복 작업 화면으로 다시 열 수 있어야 합니다.
- 속성 누락 메모를 별도 목록으로 확인할 수 있어야 합니다.
- 템플릿으로 만든 메모는 기본 제목, 본문 섹션, 속성을 함께 가져야 합니다.

## 회귀 방지 기준

- 기존 제목/본문 저장 흐름은 유지합니다.
- 서버 공유/비공유 상태는 속성 값이 아니라 기존 공유 필드 기준을 유지합니다.
- 암호화 메모의 본문은 속성 목록 검색에서 내용 노출을 확장하지 않습니다.
- 삭제 보관함의 메모는 속성 목록 대상에서 제외합니다.
