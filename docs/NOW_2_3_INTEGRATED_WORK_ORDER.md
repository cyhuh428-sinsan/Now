### 작업 분리 기준

서버/API, Web/설치형, 앱은 같은 저장소에 있지만 서로 다른 실행 산출물이다. 한 영역 수정 후에는 해당 영역 검증을 먼저 통과시킨 뒤 push한다. 다른 영역의 미커밋 변경이 남아 있으면 섞어서 커밋하지 않는다.

### Git 기준

- 수정 이후에는 반드시 커밋하고 `origin/main`에 push한다.
- exe/apk를 빌드한 경우 GitHub Release asset으로 별도 업로드한다.
- 운영 서버 테스트는 서버에서 `git pull origin main` 후 수행한다.

## 서버 및 API 작업지시

### 서버/API 목표

서버는 2.3에서 Web, 설치형 프로그램, 앱이 각자 다른 접속 방식을 사용해도 혼동 없이 운영되도록 인증, 동기화, 메신저, 첨부 저장소, 운영 점검 API를 안정화한다.

### 서버/API 현재 상태

- Web 로그인 API와 Web 세션 API는 유지되어 있다.
- 앱/설치형 접속 토큰 확인 API `/api/v1/auth/token-login`은 유지되어 있다.
- 앱/설치형 데이터 API의 `X-Now-User-Token` 검증은 유지되어 있다.
- `NOW_API_TOKEN`은 2.3 기본 사용자 흐름이 아니라 구형 개인 서버 호환/운영 보호용으로 유지한다.
- 서버 인증 역할 정리와 preflight 문서 검증은 1차 반영되었다.

### 서버/API 남은 작업

1. 운영 서버 `https://nownote.sinsan.kr`에 최신 `main`을 pull한다.
2. `server/scripts/deploy_local.sh --base-url https://nownote.sinsan.kr` 배포를 통과시킨다.
3. `/api/v1/auth/web-login`, `/api/v1/auth/web-session`, `/api/v1/auth/token-login`을 운영 서버에서 확인한다.
4. 앱/설치형 접속 토큰 실패 사유가 `user token required`, `user not found`, `invalid user token`, `device inactive`, `device_id required`처럼 구분되는지 확인한다.
5. 메신저 2.3 확장을 위한 room, participant, message, attachment API를 설계하고 구현한다.
6. 첨부 파일 저장소 경로, 파일 크기 제한, 확장자/MIME 제한, 다운로드 권한 검증을 구현한다.
7. 운영 화면/API에서 메신저 저장소, 첨부 용량, 실패 상태, 비활성 기기 상태를 확인할 수 있게 한다.

### 서버/API 완료 조건

- 서버 preflight가 통과한다.
- 운영 서버 배포가 성공한다.
- Web 로그인과 Web 세션이 정상 동작한다.
- 앱/설치형 접속 토큰 검증이 정상 동작한다.
- 앱/설치형 데이터 API가 사용자별 데이터 격리를 유지한다.
- 메신저 room/participant/message/attachment API가 권한 검증을 통과한다.
- 운영 화면/API에서 2.3 상태를 확인할 수 있다.

## Web 및 설치형 프로그램 작업지시

### Web/설치형 목표

Web과 설치형 프로그램은 2.3에서 메모앱답게 가볍고 안정적으로 동작해야 한다. 설치형 프로그램은 별도 exe 프로그램으로 유지하며, Web 전용 기능을 제외한 디자인과 기능 흐름은 Web과 맞춘다.

### Web/설치형 현재 상태

- Web은 사용자 ID/비밀번호 로그인 기준을 유지한다.
- 설치형 프로그램은 ID/password 로그인이 아니라 서버 API 연결 기준을 유지한다.
- 설치형 기본 연결값은 서버 주소, 사용자 ID, 앱/설치형 접속 토큰, 2단계 인증 코드다.
- 구형 개인 서버 API 토큰과 기기 ID는 고급 설정으로 분리되었다.
- Web/설치형 표면 검증과 정책 검증은 1차 통과했다.

### Web/설치형 남은 작업

1. 운영 서버 Web에서 로그인, 계정 생성, 비밀번호 재설정, 앱/설치형 접속 토큰 발급을 실제 확인한다.
2. 설치형에서 기존 실행 프로세스를 종료한 뒤 `npm run check:storage`를 재검증한다.
3. 설치형에서 `https://nownote.sinsan.kr` + 사용자 ID + 앱/설치형 접속 토큰으로 연결 테스트를 수행한다.
4. 제목, 본문, 전체 검색, 본문찾기 입력이 Web/설치형에서 모두 안정적인지 재확인한다.
5. `Ctrl+F`, `Ctrl+Shift+F`, Tab, Shift+Tab, Tab 간격 설정을 실제 검증한다.
6. 작은 창에서 버튼이 본문 영역을 과도하게 차지하지 않도록 아이콘/메뉴 구조를 유지한다.
7. Web idle 상태에서 반복 렌더링, 과도한 polling, 메신저 갱신으로 시스템이 느려지는 문제가 없는지 확인한다.
8. Web 메신저 2.3 확장 UI를 구현한다.
9. 설치형에는 Web 세션 전용 그룹 공유/메신저 확장 기능을 그대로 노출하지 않는다.

### Web/설치형 완료 조건

- Web이 운영 서버에서 가볍게 동작한다.
- 설치형이 별도 exe 프로그램으로 동작한다.
- 작은 창에서도 제목과 본문 편집이 가능하다.
- 검색과 본문찾기 입력이 한 글자만 입력되는 문제가 없다.
- 단축키와 Tab 들여쓰기가 기준대로 동작한다.
- Web 전용 기능은 설치형에서 숨겨져 있다.
- 설치형 서버 연결 테스트가 운영 서버 기준으로 통과한다.

### 앱 현재 상태

- 앱 서버 설정 문구는 앱/설치형 접속 토큰 중심으로 1차 정리되었다.
- 구형 개인 서버 API 토큰은 고급 설정으로 이동하는 방향으로 반영되었다.
- 첫 동기화에서 로컬 업로드 항목이 없어도 서버 조회 요청을 보내도록 1차 수정되었다.
- 앱 작업 로그와 앱 서버 설정 파일에 미커밋 변경이 남아 있다.

### 앱 남은 작업

1. 남아 있는 앱 변경을 검토한다.
2. 앱 변경이 의도한 수정인지 확인한 뒤 모바일 정적 검증을 실행한다.
3. `flutter test test/services/server_sync_service_test.dart`를 실행한다.
4. 가능하면 전체 `flutter test`를 실행한다.
5. 에뮬레이터에서 최신 debug APK를 설치하고 실행한다.
6. 앱 설정에서 `https://nownote.sinsan.kr` + 사용자 ID + 앱/설치형 접속 토큰으로 연결 테스트를 수행한다.
7. 서버에 이미 있는 일자별 메모와 계층 메모가 새 앱 설치 상태에서 내려오는지 확인한다.
8. 앱이 Web 세션 전용 그룹 공유 문서를 기대하지 않는지 확인한다.
9. 검증이 끝나면 앱 변경을 별도 커밋으로 묶고 push한다.

### 앱 완료 조건

- 앱 서버 설정 화면이 설치형 기준과 맞다.
- 앱에서 구형 개인 서버 API 토큰은 기본 흐름에서 숨겨져 있다.
- 2단계 인증 코드는 저장하지 않고 연결 테스트 때만 사용한다.
- 모바일 정적 검증이 통과한다.
- Flutter 서비스 테스트가 통과한다.
- 에뮬레이터에서 앱 실행과 서버 연결 테스트가 통과한다.
- 운영 서버에서 기존 메모 조회가 정상 동작한다.

## 2.3 핵심 목표

2.3은 메신저 기능만 추가하는 버전이 아니다.

2.3의 핵심 목표는 아래 순서다.

1. Web과 설치형 프로그램을 메모앱답게 가볍고 안정적으로 만든다.
2. 제목, 본문, 검색, 본문찾기, 단축키, Tab 들여쓰기를 안정화한다.

4. 인증/서버 연결 구조를 사용자 화면에서 단순하게 정리한다.
5. 앱 서버 설정과 조회 문제를 설치형 기준에 맞춰 정리한다.
6. 그 다음 메신저 첨부, 사진, 일부 그룹원 채팅방 기능을 추가한다.
7. 서버, Web, 설치형, 앱을 분리 검증한 뒤 같은 버전으로 릴리즈한다.

## 작업 원칙

- 메모 입력 안정화가 모든 기능보다 우선이다.

- 설치형은 Web 전용 기능을 제외하고 Web과 같은 디자인/기능 흐름을 유지한다.
- 앱은 Web/설치형 수정이 자동 반영되지 않는 별도 Flutter 앱으로 본다.
- Web 세션 전용 기능을 설치형/앱에 그대로 넣지 않는다.
- 운영자가 CLI나 DB를 직접 만져야만 확인 가능한 기능은 완료로 보지 않는다.

## 최종 완료 기준

2.3은 아래 조건을 모두 만족해야 완료다.

- Web이 가볍게 동작한다.
- 설치형 프로그램이 작은 창에서도 깨지지 않는다.
- 제목, 본문, 검색, 본문찾기 입력이 안정적이다.
- `Ctrl+F`, `Ctrl+Shift+F`, Tab 들여쓰기가 기준대로 동작한다.
- Web은 ID/password 로그인 기준을 유지한다.
- 설치형/앱은 앱/설치형 접속 토큰 기반 API 연결 기준을 유지한다.
- 구형 개인 서버 API 토큰은 기본 화면에서 숨겨져 있다.
- 앱 조회 문제 원인이 확인되고 필요한 수정이 반영되어 있다.
- 전체 그룹 메신저가 유지된다.
- 일부 그룹원 채팅방과 파일/사진 첨부가 동작한다.
- 첨부 보안 정책이 적용된다.
- 운영 화면/API로 저장소와 이상 상태를 확인할 수 있다.
- 서버/Web/설치형/앱 검증이 통과한다.
- exe와 apk가 GitHub Release asset으로 업로드되어 있다.
