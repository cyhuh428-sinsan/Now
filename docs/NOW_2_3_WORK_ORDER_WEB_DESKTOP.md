# NowNote 2.3 Web 및 설치형 프로그램 작업지시서

## 목적

Web과 설치형 프로그램은 NowNote 2.3에서 메모앱답게 가볍고 안정적으로 동작해야 한다. 설치형 프로그램은 별도 exe 프로그램이며, Web 전용 기능을 제외한 디자인과 기능 흐름은 Web과 맞춘다.

## 담당 범위

- Web 로그인 화면과 Web 세션 흐름
- Web 메모 입력, 제목 입력, 검색, 본문찾기
- Web 단축키와 Tab 들여쓰기
- Web 작은 창 레이아웃과 아이콘/메뉴 구조
- Web idle 성능과 polling 정리
- Web 메신저 UI
- 설치형 서버 API 연결 화면
- 설치형 메모 입력, 검색, 본문찾기, 단축키
- 설치형 작은 창 레이아웃
- 설치형 exe 빌드
- Web/설치형 README와 도움말

## 현재 확인된 상태

- Web은 사용자 ID/비밀번호 로그인 기준 유지
- 설치형은 ID/password 로그인이 아니라 서버 API 연결 기준 유지
- 설치형 기본 연결값은 서버 주소, 사용자 ID, 앱/설치형 접속 토큰, 2단계 인증 코드
- 구형 개인 서버 API 토큰과 기기 ID는 고급 설정으로 분리됨
- Web/설치형 표면 검증과 정책 검증 1차 통과 기록 있음
- 사용자 확인 기준으로 Web 로그인, 계정 생성, 비밀번호 재설정, 토큰 발급, 설치형 실제 연결은 이전에 성공 확인됨

## 남은 작업

1. Web에서 제목 입력, 본문 입력, 전체 검색, 본문찾기 입력을 재확인한다.
2. 설치형에서 제목 입력, 본문 입력, 전체 검색, 본문찾기 입력을 재확인한다.
3. Web/설치형에서 단축키를 확인한다.
   - `Ctrl+F`: 전체 검색
   - `Ctrl+Shift+F`: 현재 메모 본문찾기
   - `Tab`: 설정된 칸 수만큼 들여쓰기
   - `Shift+Tab`: 내어쓰기
4. 작은 창에서 상단 버튼이 본문 편집 영역을 과도하게 차지하지 않는지 확인한다.
5. Web idle 상태에서 반복 렌더링, 과도한 polling, 메신저 갱신이 시스템을 느리게 하지 않는지 확인한다.
6. Web에서 전체 그룹 채팅방 UI를 유지한다.
7. Web에서 일부 그룹원 채팅방 생성 UI를 구현한다.
8. Web에서 unread count를 표시한다.
9. Web에서 메시지 자동 갱신을 가볍게 만든다.
10. Web에서 파일/사진 첨부 선택, 이미지 미리보기, 첨부 다운로드를 구현한다.
11. Web에서 서버 첨부 정책과 업로드 실패 메시지를 표시한다.
12. 설치형에는 Web 세션 전용 그룹 공유/메신저 확장 기능을 그대로 노출하지 않는다.
13. Web/설치형 README와 도움말을 2.3 기준으로 갱신한다.
14. 설치형 최종 빌드 파일명을 `NowNote-Setup-2.3.x-x64.exe` 형식으로 맞춘다.
15. 설치형 최종 빌드 후 `npm run check:storage`를 다시 실행한다.
16. 설치형 exe를 GitHub Release asset 업로드 대상에 포함한다.

## 완료 조건

- Web/설치형 모두 입력 포커스가 끊기지 않는다.
- 검색과 본문찾기가 한 글자만 입력되는 문제가 없다.
- 단축키와 Tab 들여쓰기가 기준대로 동작한다.
- 작은 창에서도 메모 편집이 가능하다.
- Web idle 상태에서 시스템 전체가 느려지지 않는다.
- Web 전체 그룹 메신저 기존 동작이 유지된다.
- Web 일부 그룹원 채팅방을 만들 수 있다.
- Web 파일/사진 첨부가 정책 안에서 동작한다.
- 메신저가 닫혀 있을 때 시스템 성능에 영향을 주지 않는다.
- 설치형은 별도 exe 프로그램으로 생성된다.
- 설치형 exe가 2.3.x 이름으로 생성된다.
- 설치형 exe가 GitHub Release asset에 업로드된다.

## 어울 확인 항목

- Web/설치형 작업 결과 문서 확인
- Web/설치형 검증 스크립트 결과 확인
- 설치형 exe 파일명, 생성 시각, 크기 확인
- GitHub Release asset 업로드 확인
- 통합 작업지시서 완료 조건 반영
