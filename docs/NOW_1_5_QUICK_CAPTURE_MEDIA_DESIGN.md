# NowNote 1.5 첨부/미디어/빠른 기록 설계서

기준일: 2026-06-01

이 문서는 NowNote 1.5 완료 여부를 판단하는 기준 문서입니다.

## 목표

Google Keep의 빠른 캡처 장점과 Obsidian의 첨부 연결 장점을 NowNote 방식으로 흡수합니다.
지식 메모 본문을 복잡하게 만들지 않고, 빠르게 남기는 기록과 미디어/그림은 별도 기록 카드로 관리합니다.

## 포함 범위

- 빠른 캡처 카드
- 핀 고정
- 색상 표시
- 체크리스트 카드
- 시간 기반 리마인더
- 라벨/태그 빠른 적용
- 보관함 흐름
- 첨부 파일 메타데이터 기록
- 간단한 그림 카드 저장
- 기록 카드 목록 필터: 전체, 핀, 리마인더, 보관함
- Web과 설치형에서 같은 `state.data.captures` 구조 사용

## 제외 범위

- 대용량 첨부 원본 파일 저장소
- 서버 첨부 업로드/공유
- 이미지/오디오/비디오/PDF 전문 뷰어
- 모바일 빠른 기록 동등화
- 위치 기반 리마인더
- Google 계정/Calendar/Tasks 연동

제외 사유:

- 1.5에서는 기록 카드와 미디어 연결의 제품 흐름을 먼저 안정화합니다.
- 원본 파일 저장소, 서버 공유, 모바일 동등화는 별도 저장 정책과 권한 정책이 필요합니다.

## 데이터 기준

빠른 기록은 `state.data.captures` 배열에 저장합니다.

- `id`
- `content`
- `checklist`
- `labels`
- `color`
- `pinned`
- `reminderAt`
- `archived`
- `attachments`
- `sketchData`
- `createdAt`
- `updatedAt`

첨부는 1.5에서 원본 파일을 영구 보관하지 않고 이름, 유형, 크기, 임시 미리보기 데이터만 기록합니다.
그림은 간단한 Canvas 그림을 PNG data URL로 저장합니다.

## 화면 기준

- 사용자는 빠른 기록 버튼으로 기록 화면을 열 수 있어야 합니다.
- 제목 없이 내용을 저장할 수 있어야 합니다.
- 체크리스트로 저장할 수 있어야 합니다.
- 색상, 핀, 라벨, 리마인더를 카드에 적용할 수 있어야 합니다.
- 첨부 파일을 선택하면 파일 이름, 유형, 용량이 카드에 표시되어야 합니다.
- 그림판에 간단히 그리고 카드에 함께 저장할 수 있어야 합니다.
- 보관한 카드는 기본 목록에서 사라지고 보관함 필터에서 보여야 합니다.

## 회귀 방지 기준

- 지식 메모 본문 저장 구조는 변경하지 않습니다.
- 빠른 기록 카드는 삭제 보관함, 그래프, 속성, Canvas 데이터와 분리합니다.
- 서버 첨부 업로드가 구현되지 않았으므로 첨부 원본 공유 완료처럼 표시하지 않습니다.
- 리마인더는 1.5에서 화면 기반 다시 보기 기준이며 운영 서버 알림으로 확장하지 않습니다.
