# NowNote 오픈소스 라이선스 선택 가이드

이 문서는 NowNote를 공개 저장소로 열기 전에 라이선스를 선택하기 위한 판단 기준입니다.
라이선스는 법적 선택이므로 자동으로 정하지 않고, 신산님이 최종 결정한 뒤 루트에 `LICENSE` 파일을 추가합니다.

## 확정 결과

- 선택 라이선스: Apache License 2.0
- 확정일: 2026-05-29
- 적용 범위: 저장소 전체
- 확정 사유: 공개 사용성을 열어두되 특허 조항까지 명확히 하기 위함
- 적용 파일: 루트 `LICENSE`

이 결정은 신산님이 확정한 현재 공개 기준이며, 이후 변경이 필요하면 별도 결정으로 다룹니다.

## 선택 전에 정할 것

- 다른 사람이 NowNote 서버를 수정해서 자기 서버로 운영해도 되는가?
- 수정한 서버를 공개하지 않아도 되는가, 아니면 공개 의무를 두고 싶은가?
- 기업이나 다른 프로젝트가 NowNote 코드를 제품에 넣어도 되는가?
- 특허 관련 보호 문구가 필요한가?
- 외부 기여를 받을 때 기여 코드도 같은 라이선스로 받을 것인가?

## 후보 1. MIT License

가장 단순한 선택지입니다.

- 장점: 사용, 수정, 배포가 단순하고 진입 장벽이 낮음
- 장점: 개인 개발자와 작은 프로젝트가 사용하기 쉬움
- 주의: 누군가 NowNote 서버를 수정해 별도 서비스로 운영해도 수정 코드 공개 의무가 약함
- NowNote에 맞는 경우: 최대한 많이 쓰이게 하는 것이 우선일 때

## 후보 2. Apache License 2.0

MIT보다 조금 더 명확한 기업 친화형 선택지입니다.

- 장점: 특허 관련 조항이 포함되어 있음
- 장점: 기업이나 조직이 도입할 때 검토하기 쉬운 편
- 주의: MIT보다 문서가 길고 단순함은 약간 줄어듦
- NowNote에 맞는 경우: 공개 사용성을 열어두되 특허 조항까지 명확히 하고 싶을 때

## 후보 3. AGPLv3

서버형 프로그램의 공개 의무를 강하게 두는 선택지입니다.

- 장점: 누군가 수정한 NowNote 서버를 네트워크 서비스로 제공하면 수정 소스 공개 의무가 생김
- 장점: 공용 서버형 지식 수집 구조의 공개성을 강하게 지킬 수 있음
- 주의: 기업이나 일부 사용자에게 부담이 커질 수 있음
- NowNote에 맞는 경우: 서버를 고쳐서 서비스하는 쪽도 수정 내용을 공개하게 하고 싶을 때

## 추천 판단 흐름

1. 널리 쓰이는 것이 가장 중요하면 MIT를 검토합니다.
2. 널리 쓰이되 특허 조항까지 명확히 하고 싶으면 Apache 2.0을 검토합니다.
3. 서버 수정본을 비공개 서비스로 운영하는 것을 막고 싶으면 AGPLv3를 검토합니다.

## NowNote 관점 메모

- 모바일 앱, Web/설치형, Docker 서버가 한 저장소에 같이 있으므로 라이선스는 저장소 전체에 적용하는 것이 가장 단순합니다.
- 서버를 공용으로 운영할 계획이 있으므로, 서버 수정본 공개 의무를 둘지 여부가 핵심 판단입니다.
- 외부 기여를 받을 예정이라면 `CONTRIBUTING.md`에 기여 코드가 선택한 라이선스로 제공된다는 문구를 추가하는 것이 좋습니다.
- 라이선스는 Apache License 2.0으로 확정했고, 루트에 `LICENSE` 파일을 추가했습니다.
