# NowNote 2.3 그룹 메신저 미확인 표시 설계서

## 목적

2.3의 목표는 2.2 그룹 메신저에 미확인 메시지 수와 읽음 처리를 추가해, 사용자가 새 그룹 대화를 놓치지 않게 하는 것이다.

외부 push 알림, 모바일 알림, 메일 알림은 2.3 범위가 아니다. 2.3은 hosted Web 화면 안에서 확인 가능한 운영형 표시만 다룬다.

## 적용 범위

| 구분 | 내용 |
| --- | --- |
| 제공 위치 | 서버 hosted Web |
| 사용자 범위 | 같은 `group_name`을 가진 활성 사용자 |
| 권한 기준 | Web 로그인 세션 필요 |
| 저장 단위 | 사용자별 그룹 메시지 읽음 위치 |
| 제외 | push 알림, 개인 DM, 읽음 사용자 목록, 모바일/설치형 알림 |

## 사용자 흐름

1. 사용자가 hosted Web에 로그인한다.
2. 그룹 메신저 버튼에 미확인 메시지 수가 표시된다.
3. 그룹 메신저를 열면 최근 메시지를 조회한다.
4. 열린 메시지 목록의 마지막 메시지를 읽음 기준으로 저장한다.
5. 이후 같은 사용자에게는 이미 읽은 메시지를 미확인으로 다시 표시하지 않는다.

## 운영 기준

- 미확인 수는 같은 그룹 메시지 중 본인이 보낸 메시지를 제외하고 계산한다.
- 읽음 상태는 사용자와 그룹 단위로 저장한다.
- 다른 그룹의 메시지나 읽음 상태는 조회/수정할 수 없다.
- Web 세션이 없으면 읽음 상태 API를 사용할 수 없다.

## 데이터 구조

`group_message_reads`

| 필드 | 설명 |
| --- | --- |
| `owner_id` | 읽음 상태를 가진 사용자 ID |
| `group_name` | 사용자 그룹 |
| `last_read_message_id` | 마지막으로 읽은 그룹 메시지 ID |
| `read_at` | 읽음 처리 시각 |

## 완료 기준

- hosted Web의 그룹 메신저 버튼에 미확인 메시지 수가 표시된다.
- 그룹 메신저 목록 조회 API가 `unread_count`, `last_read_message_id`를 반환한다.
- 그룹 메신저를 열면 읽음 상태가 저장된다.
- 같은 그룹의 다른 사용자가 보낸 새 메시지만 미확인 수에 포함된다.
- 서버 capability에 `group_messenger_unread`가 표시된다.
- 기존 2.2 메시지 조회/전송, 2.1 그룹 공유 조회, 설치형/모바일 동작은 변경되지 않는다.
