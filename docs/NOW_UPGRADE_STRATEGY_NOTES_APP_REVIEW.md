# NowNote 메모 기능 업그레이드 전략

기준일: 2026-05-31

이 문서는 Obsidian의 지식 메모 기능과 NowNote의 현재 기능을 비교해 향후 업그레이드 전략을 정리한 문서입니다.
NowNote 1.1의 중심은 설치형 PC 클라이언트 분리입니다.
이 문서는 그 이후 자체 메모 기능이 약해지지 않도록 1.1부터 2.0까지의 단계별 업그레이드 전략을 정리합니다.
2.0의 목표는 Obsidian과 Google Keep에서 NowNote에 꼭 필요하고 유용한 기능은 더 이상 가져올 것이 없을 정도로 선별 흡수하는 것입니다.

## 문서 지위

- 이 문서는 1.1 완료 기준 문서는 아닙니다.
- 1.1부터 2.0까지 기능 확장 우선순위를 판단하는 전략 문서입니다.
- 실제 구현 여부는 각 버전 설계서에서 다시 확정합니다.
- 큰 구조 변경은 이 문서만으로 확정하지 않고 별도 설계서에서 확정합니다.

## 비교 기준

Obsidian의 강점은 단순 Markdown 편집기가 아니라 노트 간 관계를 계속 드러내는 데 있습니다.
공식 기능 기준으로 보면 핵심 축은 다음과 같습니다.

- 그래프뷰: 전체 vault의 노트와 내부 링크를 노드/엣지로 보여주고, 필터, 그룹, 방향, 노드 크기, 로컬 그래프를 제공합니다.
- 백링크: 현재 노트를 참조하는 linked mentions와 unlinked mentions를 보여줍니다.
- Canvas: 무한 2D 공간에서 노트, 텍스트 카드, 미디어, 웹페이지를 배치하고 선으로 연결합니다.
- Bases/Properties: 노트 속성, 파일 속성, 필터, 수식, 테이블 뷰로 노트를 데이터처럼 관리합니다.
- 그 외 핵심 플러그인: 북마크, 파일 복구, 포맷 변환, 노트 합치기/나누기, 나가는 링크, 목차, 링크 미리보기, Publish, 빠른 전환, 랜덤 노트, 검색, 슬래시 명령, 슬라이드, Sync, 태그, 템플릿, 고유 노트 생성, Web viewer, 단어 수, 작업공간 저장이 있습니다.

NowNote는 현재 검색, 태그, 백링크, `[[메모 제목]]` 연결 보기, 계층 메모, 서버 분석 결과 붙이기 기능이 있습니다.
하지만 전체 지식 구조를 시각적으로 조망하거나, 관계를 운영 가능한 데이터로 관리하는 기능은 아직 약합니다.

## Google Keep 비교 기준

Google Keep의 강점은 깊은 지식 관리가 아니라 빠른 캡처와 가벼운 정리입니다.
공식 도움말 기준으로 보면 핵심 축은 다음과 같습니다.

- 빠른 노트 작성, 목록 작성, 그림/이미지/오디오 기록
- 라벨, 색상, 핀 고정으로 빠르게 분류
- 보관 처리와 삭제
- 시간 기반 리마인더
- 공유와 공동 편집
- 다른 앱으로 보내기

NowNote에 가져올 것은 “빠른 캡처를 놓치지 않게 하는 기능”입니다.
Google Keep의 모든 기능이나 Google 생태계 의존 기능을 그대로 복제하지 않습니다.

## Google Keep에서 가져올 기능

- 핀 고정: 중요한 일자별 메모, 지식 메모, 기록 카드를 상단에 고정합니다.
- 색상 표시: 메모의 성격과 긴급도를 빠르게 구분합니다.
- 체크리스트: 간단한 할 일과 점검 목록을 메모 안에서 다룹니다.
- 빠른 캡처 카드: 제목 없이 바로 쓰고 나중에 정리할 수 있는 임시 기록함을 둡니다.
- 리마인더: 날짜/시간 기준 알림과 다시 보기 목록을 제공합니다.
- 라벨/태그 빠른 적용: 모바일과 설치형에서 한 번에 분류합니다.
- 보관함 흐름: 삭제가 아니라 화면에서 치우는 archive 기준을 명확히 합니다.
- 공유 메모 협업 1차: 서버 공유 문서에서 공동 편집 또는 공유 상태 확인 흐름을 검토합니다.

## Google Keep에서 가져오지 않을 기능

- Google Tasks, Calendar, Docs에 직접 의존하는 구조
- 위치 기반 리마인더
- Google 계정 기반 협업 모델
- 그림판 수준의 자유 드로잉
- Keep과 같은 단일 보드형 UI 전체 복제

제외 이유:
- NowNote는 로컬 우선, 서버 선택형, 설치형 중심 제품입니다.
- 위치 기반 알림과 Google 생태계 의존은 운영 범위가 커집니다.
- 자유 드로잉은 Canvas와 기록 모듈이 안정된 뒤 다시 검토하는 편이 안전합니다.

## NowNote 현재 강점

- 한국어 중심 사용 흐름과 운영 화면 기준이 명확합니다.
- PC 설치형, Web, 모바일, 서버를 한 제품 흐름으로 묶고 있습니다.
- 일자별 메모, 계층 메모, 음성 메모, 서버 동기화가 이미 연결되어 있습니다.
- 메모 단위 암호화와 서버 공유/비공유 기준이 있습니다.
- 서버 분석 작업 큐와 결과를 메모에 반영하는 기초 흐름이 있습니다.
- Web/설치형 화면에서 검색, 태그, 백링크, 연결 보기, 탭, 삭제 보관함, Markdown 작성 기능이 있습니다.

## NowNote 약한 부분

- 전체 그래프뷰가 없습니다.
- 현재 연결 보기는 선택 메모 주변 연결 확인에 가깝고, vault 전체 구조 조망은 약합니다.
- 로컬 그래프 깊이, 필터, 그룹, 고립 메모, 허브 메모, 최근 수정 기준 시각화가 없습니다.
- unlinked mention, 즉 제목은 언급됐지만 `[[링크]]`로 연결되지 않은 후보를 적극적으로 보여주지 않습니다.
- outgoing links, page preview, bookmarks, workspaces 같은 탐색 보조 기능이 부족합니다.
- Canvas처럼 생각을 배치하고 연결하는 자유 공간이 없습니다.
- 메모 속성 기반 테이블/필터/목록 기능이 부족합니다.
- 템플릿, 고유 노트 생성, 노트 합치기/나누기 같은 작성 보조 기능이 부족합니다.
- 파일 복구, 포맷 변환, 가져오기 보정 같은 vault 관리 기능이 아직 약합니다.
- Publish/슬라이드처럼 메모를 외부 결과물로 바꾸는 기능이 부족합니다.
- 사진/첨부/블로그형 기록은 1차 범위 밖으로 보류되어 있습니다.
- 장기 분석, 임베딩, LLM 지식화는 작업 시스템 설계가 아직 별도입니다.

## 전략 원칙

- Obsidian을 그대로 복제하지 않습니다.
- Obsidian의 유용한 핵심 기능은 2.0까지 최대한 흡수합니다.
- NowNote는 운영형 시스템이므로 화면, 상태, 동기화, 백업, 장애 표시까지 함께 설계합니다.
- 그래프뷰는 예쁜 장식이 아니라 누락, 고립, 연결 과다, 오래된 지식, 공유 상태를 찾는 운영 도구로 설계합니다.
- Canvas는 그래프와 속성 관리가 안정된 뒤 1.x 후반 또는 2.0에서 구현합니다.
- 사진/첨부는 일반 메모 본문에 억지로 넣지 않고 기록/블로그형 모듈 후보로 둡니다.
- LLM/임베딩은 사용자가 브라우저를 열어두지 않아도 며칠 동안 진행 가능한 서버 작업 시스템으로 설계합니다.
- 2.0 완료 시점에는 Obsidian/Google Keep 대비 남은 차이가 “NowNote 제품 방향상 의도적으로 제외한 것”이어야 합니다.

NowNote 1.x는 한 번에 1.5나 2.0으로 뛰지 않고 아래 순서로 단계화합니다.
1.1부터 1.9까지 기능군을 세분화해 흡수하고, 2.0에서는 최종 누락 점검과 LLM 지식화를 마무리합니다.

### 1.1: 설치형 PC 클라이언트 분리 우선

목표:
- PC 로컬 원본과 서버 공유 문서를 분리합니다.
- 향후 그래프와 지식 기능이 로컬 원본을 기준으로 안전하게 동작할 기반을 만듭니다.

포함:
- 설치형 로컬 파일 저장소
- 공유/비공유 상태 분리
- 자동 동기화와 충돌 처리
- 암호화 메모와 삭제 보관함 안정화

포함하지 않음:
- 전체 그래프뷰 본구현
- Canvas
- 임베딩 기반 지식화
- 사진/첨부 본구현
- Publish/슬라이드/템플릿/작업공간 같은 Obsidian 확장 기능 본구현

이유:
- 저장 원본이 불안정한 상태에서 그래프뷰를 먼저 만들면 Web 공유 문서, 설치형 로컬 문서, 서버 문서의 기준이 섞입니다.
- 1.1에서 먼저 “어떤 문서가 원본인가”를 확정해야 이후 그래프, 속성, 분석이 흔들리지 않습니다.

### 1.2: 관계 탐색과 그래프뷰

목표:
- Obsidian의 그래프뷰와 백링크 강점을 NowNote 방식으로 흡수합니다.
- 메모 간 관계를 사용자가 직접 확인하고 정리할 수 있게 합니다.

주요 기능:
- 전체 그래프뷰
- 선택 메모 중심 로컬 그래프
- 그래프 필터: 검색어, 태그, 즐겨찾기, 공유 상태, 암호화 여부, 삭제 제외, 최근 수정
- 그래프 그룹: 주제, 태그, 공유 상태, 분석 상태
- 고립 메모 목록
- 허브 메모 목록
- outgoing links 패널
- 링크 미리보기
- 연결 후보: unlinked mention 후보 표시
- 링크 제안: 제목이 본문에 등장하지만 연결되지 않은 메모 제안
- 그래프에서 메모 열기, 공유 전환, 즐겨찾기 전환
- 북마크 1차: 메모, 검색어, 그래프 필터 저장

완료 기준 후보:
- 전체 그래프에서 노트와 `[[링크]]`가 노드/엣지로 표시됩니다.
- 로컬 그래프에서 깊이 1-3을 선택할 수 있습니다.
- 고립 메모와 허브 메모를 화면에서 확인할 수 있습니다.
- 연결 후보를 보고 `[[링크]]`로 반영할 수 있습니다.
- 현재 메모의 나가는 링크와 링크 미리보기를 확인할 수 있습니다.
- 자주 쓰는 메모와 검색 조건을 북마크로 저장할 수 있습니다.
- Web은 서버 공유 문서 기준 그래프만 표시하고, 설치형은 PC 로컬 원본 기준 그래프를 표시합니다.

### 1.3: 속성 기반 지식 관리

목표:
- 메모를 단순 문서가 아니라 관리 가능한 지식 항목으로 다룹니다.
- Obsidian Bases/Properties와 비슷한 역할을 NowNote 운영형 화면으로 구현합니다.

주요 기능:
- 메모 속성: 상태, 중요도, 유형, 출처, 작성자, 마감일, 관련 프로젝트
- 속성 편집 패널
- 속성 기반 목록 뷰
- 필터 저장
- 정렬과 그룹화
- 누락 속성 점검
- 공유 상태, 암호화 여부, 분석 여부를 속성처럼 확인
- 템플릿 1차: 속성과 기본 제목/섹션을 가진 새 메모 템플릿
- 고유 노트 생성: 시각 기반 제목 또는 ID로 빠른 메모 생성
- 노트 합치기/나누기 1차
- 속성 기반 일자별 메모 연결

완료 기준 후보:
- 사용자가 메모 속성을 화면에서 추가/수정할 수 있습니다.
- 저장된 필터 뷰를 만들고 다시 열 수 있습니다.
- 속성 누락 메모를 점검 목록으로 볼 수 있습니다.
- 템플릿으로 새 메모를 만들 수 있습니다.
- 메모 합치기/나누기가 원본 백업 기준과 함께 동작합니다.
- Web과 설치형에서 같은 속성 구조를 사용하되, 표시 대상은 각 제품 기준을 따릅니다.

### 1.4: Canvas와 시각적 사고 정리

목표:
- Obsidian Canvas의 장점을 NowNote 방식으로 흡수합니다.
- 그래프가 자동 관계 조망이라면 Canvas는 사용자가 직접 생각을 배치하는 작업 공간으로 봅니다.

주요 기능:
- 무한 또는 큰 2D 보드
- 메모 카드, 텍스트 카드, 기록 카드, 웹 링크 카드
- 카드 간 선 연결
- 연결 라벨
- 그룹
- 색상
- 확대/축소, 전체 보기, 선택 영역 보기
- 그래프에서 선택한 메모 묶음을 Canvas 초안으로 만들기
- Canvas 저장, 동기화, 충돌 처리
- Canvas를 메모에 임베드하거나 링크로 연결

완료 기준 후보:
- 사용자가 메모 카드를 배치하고 선으로 연결할 수 있습니다.
- Canvas가 설치형 로컬 저장소와 서버 공유 기준을 따릅니다.
- Canvas 충돌이 일반 메모 충돌처럼 보류/해결됩니다.
- Canvas 카드가 기존 백링크/그래프 기능과 가능한 범위에서 연결됩니다.

### 1.5: 첨부, 미디어, 빠른 기록

목표:
- Google Keep의 빠른 캡처 장점과 Obsidian의 첨부 파일 지원을 NowNote 방식으로 흡수합니다.
- 사진, 첨부, 생활 기록을 계층 메모 본문에 무리하게 넣지 않고 별도 기록 모듈로 분리합니다.
- 지식 메모는 글자뿐 아니라 그림과 음성으로도 남길 수 있어야 하므로 간단한 손그림 기록을 포함합니다.

주요 기능:
- 빠른 캡처 카드
- 핀 고정
- 색상 표시
- 체크리스트
- 시간 기반 리마인더
- 라벨/태그 빠른 적용
- 보관함 흐름
- 사진/첨부 기록
- 날짜별 기록 타임라인
- 원본 파일 보관 위치 표시
- 서버 공유 여부
- 이미지, 오디오, 비디오, PDF 보기
- 간단한 그림 그리기: 펜, 지우개, 색상, 두께
- 그림 카드 저장: 지식 메모 본문에 직접 섞지 않고 링크/미리보기 카드로 연결
- 그림 원본 PNG 내보내기
- 첨부 썸네일과 용량 표시
- 첨부 누락/고아 파일 점검
- 모바일 음성 메모와 PC 기록 연결

완료 기준 후보:
- 제목 없이 빠르게 기록하고 나중에 정리할 수 있습니다.
- 핀, 색상, 체크리스트, 리마인더가 화면에서 동작합니다.
- 사진/첨부가 일반 지식 메모 저장 흐름을 깨지 않습니다.
- 사용자가 설치형 또는 모바일에서 간단한 그림을 그리고 메모에 연결할 수 있습니다.
- 그림은 지식 메모 본문 원문을 깨지 않는 별도 카드/첨부로 저장됩니다.
- 원본 파일 위치와 서버 전송 여부를 화면에서 확인할 수 있습니다.
- 기록을 날짜, 태그, 색상, 핀, 공개 상태로 필터링할 수 있습니다.
- 첨부 누락/고아 파일을 화면에서 확인할 수 있습니다.

- 사용자가 빠르게 만들고, 나누고, 합치고, 명령을 실행할 수 있게 합니다.

- 속성 포함 템플릿

- 노트 나누기
- 빠른 전환 고도화
- 랜덤 메모 열기
- 작성 중 명령 검색
- 반복 작성 패턴 저장

완료 기준 후보:

주요 기능:

- Publish형 공개 묶음 1차
- Slides형 발표 내보내기 1차
- 공개 묶음별 포함 문서 선택
- 공개 제외 속성
- permalink, description, cover 같은 공개용 속성

- 정적 사이트 내보내기
- 발표 모드

완료 기준 후보:

- 공개 묶음과 발표용 내보내기를 만들 수 있습니다.
- 공개 제외 문서와 암호화 메모가 외부 산출물에 섞이지 않습니다.
- 발표 모드가 Markdown 구분선 기준으로 동작합니다.
- 공개 전 점검에서 민감정보 후보를 확인할 수 있습니다.

- 지식 건강 점검 화면에서 정리 우선순위를 확인할 수 있습니다.

### 2.0: Obsidian 유용 기능 흡수 완료와 LLM 지식화

목표:
- Obsidian과 Google Keep에서 NowNote에 필요하고 유용한 기능은 더 이상 가져올 것이 없도록 1차 흡수를 완료합니다.
- 1.1부터 1.9까지 구현한 기능군을 통합 점검합니다.
- 단순 요약이 아니라 장기 지식 작업 시스템을 완성합니다.
- 사용자가 브라우저를 닫아도 서버가 작업을 이어가고, 결과를 검토한 뒤 메모 구조에 반영합니다.

주요 기능:
- Obsidian/Google Keep 기능 흡수 완료 점검표
- 1.1-1.9 기능 통합 회귀 점검
- 긴 작업 큐
- 진행률 표시
- 중단/재개
- 실패 재시도
- 작업 로그
- 임베딩 색인
- 유사 메모 추천
- 중복/상충 내용 후보
- 지식 구조 제안
- 관계 후보 자동 제안
- 태그/속성 후보 자동 제안
- 그래프 기반 지식 건강 점검
- Canvas 기반 생각 묶음 분석
- 결과 승인/반려 화면
- 대량 변경 전 자동 스냅샷
- 사용자 승인 후 메모 구조 반영

완료 기준 후보:
- Obsidian 핵심 플러그인과 Google Keep 기능 중 NowNote에 유용한 기능은 모두 구현됐거나 의도적 제외로 문서화됩니다.
- 며칠 걸리는 작업도 브라우저가 멈추지 않습니다.
- 작업 상태를 Web/설치형/운영 화면에서 확인할 수 있습니다.
- 실패한 작업은 재시도하거나 보류할 수 있습니다.
- LLM 결과는 자동 반영이 아니라 사용자가 승인한 뒤 메모에 반영됩니다.
- 암호화 메모는 기본적으로 분석 대상에서 제외됩니다.
- 2.0 완료 문서에 “Obsidian에서 추가로 가져올 기능 없음” 판정이 남습니다.

## 그래프뷰 설계 방향

NowNote 그래프뷰는 다음 상태를 함께 보여줘야 합니다.

- 노트 간 `[[링크]]`
- 백링크 수
- 태그
- 즐겨찾기
- 공유/비공유 상태
- 암호화 여부
- 최근 수정
- 분석 완료 여부
- 고립 여부

초기 구현은 2D force graph로 충분합니다.
단, 화면은 장식용 전면 그래픽이 아니라 메모 정리용 도구여야 합니다.
그래프에서 바로 메모 열기, 필터 적용, 고립 메모 목록 이동, 링크 후보 적용까지 이어져야 합니다.

## Canvas 검토

Canvas는 유용하지만 그래프와 속성 관리 뒤에 구현합니다.
이유는 다음과 같습니다.

- Canvas는 저장 형식, 동기화, 충돌 처리가 별도로 필요합니다.
- 모바일과 Web에서 같은 경험을 제공하기 어렵습니다.
- 그래프뷰와 속성 관리가 먼저 있어야 Canvas가 단순 그림판이 되지 않습니다.

따라서 Canvas는 1.4 후보로 둡니다.
다만 1.2 그래프뷰에서 선택한 메모 묶음을 나중에 Canvas로 보낼 수 있도록 데이터 구조는 막지 않습니다.

## 사진/첨부 재검토

사진/첨부는 초기 논의에서 필요성이 있었지만 1차 범위에서는 제외했습니다.
재검토 결과, 지식 메모 본문에 직접 섞는 방식은 적합하지 않습니다.

권장 방향:
- 별도 기록/블로그형 모듈로 분리합니다.
- 메모 본문에는 필요한 경우 기록 링크만 넣습니다.
- 첨부 원본, 썸네일, 서버 공유 상태, 공개 여부를 별도 관리합니다.

## LLM 지식화 재검토

LLM 지식화는 단순 버튼 하나가 아니라 운영형 작업 시스템입니다.
서버 분석 작업 큐가 이미 있으므로 이것을 확장하되, 결과 자동 반영은 금지하는 편이 안전합니다.

권장 방향:
- 먼저 그래프와 속성 기반 구조를 만듭니다.
- 그 다음 LLM이 관계 후보, 태그 후보, 중복 후보, 요약 후보를 제안합니다.
- 최종 반영은 사용자가 승인합니다.
- 2.0에서는 Obsidian/Google Keep 기능 흡수 완료 점검과 LLM 지식화가 함께 완료되어야 합니다.

## 1.1 설계서와의 관계

`docs/NOW_1_1_DESKTOP_CLIENT_DESIGN.md`가 1.1 완료 기준입니다.
이 문서는 1.1 이후 메모 기능 확장 전략입니다.

따라서 1.1 작업 중에는 다음만 반영합니다.

- 로컬 원본과 서버 공유 문서 분리
- 그래프/속성/LLM 확장을 막지 않는 데이터 구조
- 공유 상태, 암호화 상태, 삭제 상태를 명확히 남기는 저장 기준

그래프뷰, 속성 관리, 사진/첨부, LLM 장기 작업은 1.1 완료 조건에 넣지 않습니다.

## 2.0 완료 판정

NowNote 2.0은 다음 조건을 만족해야 완료로 봅니다.

- Obsidian 핵심 기능 중 NowNote에 유용한 기능이 모두 구현됐습니다.
- Google Keep 기능 중 NowNote에 유용한 빠른 캡처 기능이 모두 구현됐습니다.
- 구현하지 않은 Obsidian/Google Keep 기능은 의도적 제외 사유가 문서화됐습니다.
- 그래프, 백링크, outgoing links, Canvas, 속성/Bases, 템플릿, 복구, Publish형 내보내기, 작업공간, 핀, 색상, 체크리스트, 리마인더 기능이 실제 화면에서 연결됩니다.
- 설치형, Web, 모바일, 서버의 데이터 기준이 충돌하지 않습니다.
- 장기 LLM/임베딩 작업이 운영 화면과 사용자 화면에서 상태 확인 가능합니다.
- 사용자는 LLM 제안을 검토하고 승인/반려할 수 있습니다.
- 2.0 완료 문서에 “Obsidian/Google Keep에서 추가로 가져올 필요 기능 없음”이 명시됩니다.

## 다음 결정 필요 항목

- 1.2를 그래프뷰 중심으로 확정할지 여부
- 그래프뷰를 Web에도 제공할지, 설치형 우선으로 만들지 여부
- 속성 값을 Markdown frontmatter처럼 본문에 노출할지, NowNote 내부 JSON 속성으로 둘지 여부
- Canvas를 1.4에서 전체 구현할지, 1.4에서는 1차 구현만 할지 여부
- 사진/첨부 모듈을 1.5에 포함할지 여부
- Google Keep식 리마인더를 앱 내부 알림으로만 둘지, 운영 서버 알림까지 확장할지 여부
- 2.0에서 LLM 지식화가 제안만 할지, 사용자 승인 후 자동 편집까지 할지 여부
