# NowNote 2.3 앱 작업지시서

## 목적

앱은 Web/설치형과 별도 Flutter 앱으로 관리한다. 2.3에서는 앱 서버 연결과 동기화가 운영 서버 기준으로 안정적으로 동작하는지 유지하고, APK 릴리즈 산출물을 정리한다.

## 담당 범위

- 앱 서버 설정 화면
- 앱/설치형 접속 토큰 기반 연결
- 2단계 인증 코드 요청 시 입력
- 구형 개인 서버 API 토큰 고급 설정 분리
- 서버 메모 동기화
- 모바일 정적 검증
- Flutter 테스트
- APK 빌드
- 앱 README와 모바일 런타임 체크리스트

## 현재 확인된 상태

- 에뮬레이터에서 `https://nownote.sinsan.kr` 연결 성공
- 사용자 ID + 앱/설치형 접속 토큰 인증 성공
- 사용자 프로필, 그룹, 이메일, 시간대, 2단계 상태 조회 성공
- `전체 다시 동기화` 실행 성공
- 서버 변경 5건 확인
- `기록 > 메모 > 계층 메모`에서 서버 항목 표시 확인
- 모바일 표면 검증 142/142 통과
- `flutter test` 61개 통과

## 남은 작업

1. 앱 인증 연결과 서버 메모 내려오기 동작을 유지한다.
2. 모바일 메신저 확장은 이번 2.3 범위에서 제외한다.
3. APK 최종 빌드를 수행한다.
4. APK 파일명을 `NowNote-2.3.x.apk` 형식으로 맞춘다.
5. APK를 GitHub Release asset 업로드 대상에 포함한다.
6. 앱 README와 모바일 런타임 체크리스트를 2.3 기준으로 갱신한다.
7. APK 업로드 후 앱 작업 로그에 파일명, 생성 시각, 검증 결과를 기록한다.

## 완료 조건

- 앱 인증 연결과 서버 메모 내려오기가 운영 서버 기준으로 유지된다.
- 앱에서 구형 개인 서버 API 토큰은 기본 흐름에서 숨겨져 있다.
- 2단계 인증 코드는 저장하지 않고 연결 테스트 때만 사용한다.
- 모바일 정적 검증이 통과한다.
- Flutter 테스트가 통과한다.
- APK가 2.3.x 이름으로 생성된다.
- APK가 GitHub Release asset에 업로드된다.
- 앱 작업 로그에 최종 APK 산출물 정보가 남아 있다.
