# NowNote 1.6 작성 보조와 명령 체계 설계서

기준일: 2026-06-01

## 목적

NowNote 1.6의 목표는 사용자가 지식 메모를 더 빠르게 만들고 정리할 수 있도록 작성 보조와 명령 실행 흐름을 강화하는 것입니다.
Obsidian의 Templates, Unique note creator, Note composer, Command palette, Slash commands, Quick switcher, Random note 기능 중 NowNote에 꼭 필요한 기능만 흡수합니다.

## 포함 범위

- 명령 팔레트
- 작성 중 Slash command
- 기본 템플릿 메모 생성
- 속성 포함 템플릿 메모 생성
- 고유 ID/시각 기반 메모 생성
- 하위 메모 내용 병합
- 제목 섹션 기준 메모 나누기
- 빠른 전환 보조
- 랜덤 메모 열기
- 반복 작성 패턴 1차 템플릿화

## 제외 범위

- 플러그인형 사용자 스크립트 실행
- 외부 자동화 명령 실행
- 서버 작업 큐 기반 작성 자동화
- 여러 파일을 동시에 변경하는 대량 composer
- 템플릿 파일 저장소와 동기화

## 화면 기준

작성 보조는 별도 명령 팔레트 화면에서 실행합니다.
기존 빠른 전환은 메모 열기 중심으로 유지하고, 명령 팔레트는 생성/정리/작성 명령 중심으로 분리합니다.

## Slash command 기준

본문 편집 중 한 줄에 `/` 명령을 입력하고 Enter를 누르면 해당 명령을 실행합니다.

- `/time`: 현재 시간 삽입
- `/check`: 체크리스트 삽입
- `/template project`: 프로젝트 템플릿 삽입
- `/template meeting`: 회의 템플릿 삽입
- `/template source`: 자료 템플릿 삽입
- `/unique`: 고유 메모 생성
- `/random`: 랜덤 메모 열기
- `/split`: 현재 메모를 제목 섹션 기준으로 나누기
- `/merge`: 현재 메모의 하위 메모 내용을 본문에 병합

## 데이터 기준

1.6은 새로운 대형 데이터 저장소를 만들지 않습니다.
템플릿은 앱 내부 기본 템플릿으로 제공하고, 생성된 결과는 기존 `state.data.tree` 메모 구조와 `properties` 구조를 사용합니다.

## 안전 기준

- 명령 팔레트는 사용자가 명시적으로 실행한 명령만 적용합니다.
- 합치기는 하위 메모를 삭제하지 않고 선택 메모 본문에 내용을 추가하는 1차 안전 방식으로 처리합니다.
- 나누기는 현재 메모의 `## 제목` 섹션을 하위 메모로 만들고, 원문은 섹션 앞 본문만 남깁니다.
- 암호화되어 잠긴 메모에는 본문 변경 명령을 적용하지 않습니다.
- Web은 서버 공유 문서 기준, 설치형은 로컬 원본 기준의 기존 저장 정책을 그대로 따릅니다.
