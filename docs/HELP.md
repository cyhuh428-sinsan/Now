# NowNote 도움말

English version: [NowNote Help](HELP.en.md)

NowNote는 혼자 로컬에서 쓰는 메모 프로그램으로도 사용할 수 있고, 서버에 연결해서 여러 기기와 동기화하는 프로그램으로도 사용할 수 있습니다.

처음 사용하는 사람은 먼저 아래 두 가지 중 하나를 선택하면 됩니다.

## 사용 방식 선택

### 1. 단독 사용자

단독 사용자는 서버에 연결하지 않고 현재 기기 안에서만 메모를 저장합니다.

이 방식이 맞는 경우:

- 휴대폰이나 PC 한 대에서만 메모를 관리할 때
- 서버 설치, 계정, 토큰 같은 설정 없이 바로 쓰고 싶을 때
- 개인 메모를 외부 서버에 올리고 싶지 않을 때
- 테스트나 임시 기록 용도로 가볍게 쓰고 싶을 때

사용 가능한 기능:

- 일자별 메모
- 지식 메모
- 검색
- 태그
- 백링크와 연결 보기
- Markdown 작성과 미리보기
- 삭제 보관함
- 백업과 복원(Web/설치형 JSON, 모바일 DB)
- Markdown 가져오기와 내보내기
- 화면 설정
- 단축키 설정

제한되는 기능:

- 다른 기기와 자동 동기화
- 서버 백업
- 서버 기반 사용자 관리
- 서버 기반 LLM 분석 작업
- 서버 로그인 사용자 전용 암호화 저장

단독 사용자는 주기적으로 백업 또는 Markdown 내보내기를 해두는 것이 좋습니다. Web/설치형은 JSON 백업, 모바일 앱은 DB 백업을 사용합니다.

### 2. 서버 연결 사용자

서버 연결 사용자는 NowNote 서버에 접속해서 메모를 동기화합니다.

서버 연결 방식은 두 가지입니다.

- 개인 서버: 사용자가 Docker 컨테이너로 자기 서버를 직접 설치
- 공용 서버: 기존에 운영 중인 NowNote 서버에 계정을 받아 접속

이 방식이 맞는 경우:

- 휴대폰, PC, 설치형 프로그램에서 같은 메모를 보고 싶을 때
- 서버에 백업본을 남기고 싶을 때
- 사용자 계정, 기기, 접속 상태를 관리해야 할 때
- 서버 기반 분석 기능을 사용하고 싶을 때
- 로그인 기반 암호화 저장이 필요한 운영 구조를 준비하려고 할 때

사용 가능한 기능:

- 단독 사용자 기능 전체
- 서버 동기화
- 서버 백업
- 서버 백업 검증
- 사용자 프로필
- 사용자별 시간대
- 사용자 그룹
- 활성 상태 관리
- 2단계 인증 사용 여부 관리
- 기기별 동기화 이력 확인
- 서버 운영 점검 화면
- 서버 기반 녹음 파일 저장
- 서버 기반 분석 작업

주의할 점:

- 서버 주소와 API 토큰이 필요합니다.
- 앱과 설치형 프로그램에 입력한 API 토큰은 기기 보안 저장소에 저장합니다.
- 개인 서버 사용자는 `.env`의 토큰과 DB 비밀번호를 반드시 변경해야 합니다.
- 공용 서버 사용자는 운영자가 발급한 접속 정보를 사용합니다.
- 현재 1차 서버는 개인 Docker 서버 기준의 단일 API 토큰으로 시작할 수 있고, 공용 서버용 사용자별 접속 토큰과 2단계 코드 검증도 함께 제공합니다.
- 공용 서버 오픈 전 `scripts/preflight.py --public-server` 실패 항목을 모두 해소해야 합니다. 현재 실패는 사용자별 접속 토큰 강제 설정과 HTTPS/reverse proxy 같은 공개 운영 환경이 아직 남아 있음을 알려주는 안전장치입니다.
- 서버 백업 검증은 백업 파일의 스키마, 체크섬, 필수 항목, 토큰 민감정보 노출 여부를 확인합니다.
- 서버에 연결해도 보관된 일자별 메모는 삭제하지 않고 비활성 백업 상태로 유지합니다.

## 플랫폼별 기준

### 모바일 앱

모바일 앱은 빠른 입력이 중심입니다.

설치 경로:

- 정식 배포 후: Google Play 스토어에서 NowNote 설치
- 내부 테스트 기간: 운영자가 제공한 Google Play 내부 테스트 링크로 설치
- 개발/검증용 직접 실행: `now_app/README.md`와 `now_app/docs/google_play_step_by_step_ko.md` 확인

서버를 연결하지 않아도 모바일 앱은 단독으로 사용할 수 있습니다.
서버 연결 설정을 비워두면 메모는 현재 휴대폰 안에 저장됩니다.
공용 서버에 접속하려면 앱 설치 후 설정 화면의 서버 연결 항목에 운영자가 제공한 서버 주소와 접속값을 입력합니다.

권장 사용 흐름:

- 홈에서 오늘 메모를 빠르게 작성
- 음성 또는 타이핑으로 일자별 메모 추가
- 필요한 경우 계층 메모 작성
- 서버 연결 사용자는 설정에서 서버 주소와 토큰 입력
- 앱에 입력한 API 토큰은 기기 보안 저장소에 저장
- 공용 서버가 요구하면 사용자별 접속 토큰도 입력

모바일에서 중요한 기능:

- 일자별 메모
- 음성 메모
- 빠른 메모 입력
- 서버 동기화
- 연결 상태 확인
- 서버 녹음 업로드 상태 확인
- 서버 분석 작업 등록과 결과 확인

### Web / 설치형 프로그램

Web과 설치형 프로그램은 지식 메모가 중심입니다.
Windows 설치형 프로그램은 Web 화면을 Electron 앱으로 감싼 `.exe` 설치 파일로 제공합니다.
Web 프로그램은 외부 PC에서 브라우저로 접속하는 서버 제공 화면입니다.
서버를 직접 설치하지 않는 사용자는 공용 Web 프로그램 `https://nownote.sinsan.kr`에 접속합니다.
개인 서버를 운영하는 사용자는 `https://내도메인` 또는 `http://서버IP:8750`으로 접속합니다.
화면 설정의 서버 연결 항목에는 운영자가 제공한 서버 주소와 접속값을 입력합니다.

개발자가 확인하는 `web/index.html`은 사용자용 Web 프로그램 주소가 아닙니다.
사용자용 Web 프로그램은 서버 주소 자체에서 열립니다.
`/app/`은 이전 안내와 호환하기 위한 보조 주소입니다.

권장 사용 흐름:

- 주제 / 분류 / 메모 구조로 지식 정리
- Markdown으로 긴 글 작성
- 검색, 태그, 백링크, 연결 보기로 메모 탐색
- 필요할 때 일자별 메모를 팝업처럼 열어 확인
- 단축키와 기능 표시 여부를 사용자 취향에 맞게 조정

Web과 설치형에서 중요한 기능:

- 계층형 지식 메모
- 탭 기반 편집
- 본문 찾기
- Markdown 보기
- 빠른 전환
- 연결 보기
- 기능별 ON/OFF
- 단축키 변경
- Markdown 가져오기와 내보내기
- 서버 분석 작업 등록, 결과 요약 확인, 메모 본문에 추가

## 서버 연결 설정

서버 연결에 필요한 값:

- 서버 주소
- API 토큰
- 사용자별 접속 토큰: 공용 서버 또는 운영자가 요구하는 서버에서 사용
- 2단계 인증 코드: 2단계 인증을 사용하는 사용자만 입력
- 사용자 ID
- 기기 ID

### 서버를 직접 설치하지 않는 사용자

서버를 설치하지 않고 단독으로만 쓰는 경우 서버 연결 설정은 비워둡니다.

먼저 볼 경로:

```text
사용자 도움말: docs/HELP.md
모바일 앱 안내: now_app/README.md
설치형 프로그램 안내: desktop/README.md
공용 Web 프로그램: https://nownote.sinsan.kr
공용 개인정보처리방침: https://nownote.sinsan.kr/privacy
```

공용 서버에 접속만 하는 사용자는 운영자에게 받은 값을 입력합니다.

```text
서버 주소: 예) https://nownote.sinsan.kr
API 토큰: 운영자가 제공한 서버 연결용 토큰
사용자 ID: 운영자가 만든 사용자 ID
사용자별 접속 토큰: 운영자가 발급한 사용자 토큰
2단계 인증 코드: 2단계 인증을 사용하는 경우 6자리 코드
기기 ID: 앱 또는 Web/설치형 프로그램에서 자동 생성
```

입력 위치:

```text
모바일 앱: 설정 화면의 서버 연결 항목
설치형 프로그램: 화면 설정의 서버 연결 항목
Web 프로그램: https://nownote.sinsan.kr 또는 https://내도메인의 화면 설정
```

사용자는 서버의 `.env` 파일을 직접 볼 필요가 없습니다.
공용 서버 접속값은 운영자가 별도로 알려준 값만 사용합니다.

개인 Docker 서버 예시:

```text
서버 주소: http://서버주소:8750
API 토큰: 서버 .env의 NOW_API_TOKEN
사용자별 접속 토큰: 관리자 화면에서 사용자별로 발급한 토큰
2단계 인증 코드: 필요한 경우 6자리 코드
사용자 ID: local_user 또는 운영자가 발급한 사용자 ID
기기 ID: 앱 또는 설치형 프로그램에서 자동 생성
```

개인 서버를 Linux 서버에 설치한 경우, 휴대폰이나 외부 PC에서는 `localhost`를 쓰지 않습니다.
서버의 도메인 또는 서버 IP를 사용합니다.

## 사용자 프로필

서버 연결 사용자는 서버에 사용자 정보가 생성됩니다.

사용자 정보:

- owner ID
- 이메일
- 표시 이름
- 시간대
- 사용자 그룹
- 2단계 인증 사용 여부
- 활성 상태
- 최근 접속 시간

앱과 설치형 프로그램은 자기 프로필의 이메일, 표시 이름, 시간대를 수정할 수 있습니다.
모바일 앱과 Web/설치형 프로그램에서는 서버 설정 화면에서 사용자 ID를 입력한 뒤 사용자 프로필을 불러오거나 저장합니다.

관리자는 서버 관리 화면에서 사용자 그룹, 2단계 인증 사용 여부, 활성 상태를 관리합니다.
공용 서버 운영자는 사용자 접속 전에 관리 화면에서 사용자 ID를 미리 만들 수 있습니다.
비활성 사용자는 프로필 조회 외의 서버 동기화와 데이터 API 사용이 차단됩니다.
2단계 인증을 사용하는 사용자는 토큰 로그인 확인 때 6자리 인증 코드를 함께 입력해야 합니다.

## 백업과 가져오기

### Web / 설치형 JSON 백업

JSON은 Web/설치형 NowNote 전체 데이터를 백업하거나 복원할 때 사용합니다.

포함되는 내용:

- 일자별 메모
- 보관된 일자별 메모
- 지식 메모
- 삭제 보관함
- 화면 설정
- 열린 탭 설정

JSON 가져오기는 현재 데이터를 백업 파일 내용으로 교체하는 성격이 강합니다.

### 모바일 DB 백업

모바일 앱은 `.db` 파일로 전체 데이터를 백업하거나 복원합니다.

- 설정 > 데이터 관리 > 백업 내보내기에서 DB 백업 파일을 저장합니다.
- 백업 가져오기는 현재 모바일 데이터를 백업 파일 내용으로 교체합니다.
- 가져오기 후에는 앱을 다시 실행해야 합니다.

### Markdown 가져오기

Markdown 가져오기는 다른 곳에서 작성한 `.md`, `.markdown`, `.txt` 파일을 NowNote 지식 메모로 추가하는 기능입니다.

가져온 파일은 원본 파일과 연결하지 않습니다.

- 원본 파일은 그대로 둡니다.
- NowNote 내부에 새 메모를 만듭니다.
- NowNote에서 수정해도 원본 파일은 바뀌지 않습니다.
- NowNote에서 삭제해도 원본 파일은 삭제되지 않습니다.

### Markdown 내보내기

Markdown 내보내기는 사람이 읽을 수 있는 문서 형태로 지식 메모, 일자별 메모, 보관된 일자별 메모를 저장합니다.

내보내기 기준:

- 현재 화면 언어에 맞춰 한국어 또는 영어 형식으로 생성합니다.
- 지식 메모 제목에는 주제 / 분류 / 메모 단계가 표시됩니다.
- 일자별 메모와 보관된 일자별 메모도 함께 포함합니다.
- NowNote에서 내보낸 한국어 Markdown과 영어 Markdown은 모두 다시 가져올 수 있습니다.

## 암호화 저장

암호화 저장은 현재 1차 범위에서는 켜지지 않는 기능입니다.

향후 서버 로그인 사용자 전용 선택 기능으로 제공하고, 기본값은 꺼짐으로 둡니다.

원칙:

- 로그인은 사용 권한 확인에 사용합니다.
- 실제 암호화 키는 사용자 비밀번호 또는 복구키 기반으로 분리합니다.
- 암호화된 메모는 서버 운영자도 원문을 읽을 수 없어야 합니다.
- 암호화 메모는 기본적으로 LLM 분석에서 제외합니다.

이 기능은 저장/동기화 구조가 더 확정된 뒤 구현합니다.

## 추천 시작 방법

처음에는 단독 사용자로 시작해도 됩니다.

1. Web 또는 앱에서 메모 구조를 먼저 정합니다.
2. 일자별 메모와 지식 메모를 사용해 봅니다.
3. 여러 기기에서 같은 메모가 필요해지면 서버를 연결합니다.
4. 서버 연결 후 사용자 프로필과 시간대를 확인합니다.
5. 운영이 안정되면 백업, 분석, 암호화 기능을 단계적으로 켭니다.
