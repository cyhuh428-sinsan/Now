# NowNote 도움말

English version: [NowNote Help](HELP.en.md)

NowNote는 혼자 로컬에서 쓰는 메모 프로그램으로도 사용할 수 있고, 서버에 연결해서 여러 기기와 동기화하는 프로그램으로도 사용할 수 있습니다.

처음 사용하는 사람은 먼저 아래 두 가지 중 하나를 선택하면 됩니다.

## 사용 방식 선택

### 1. 단독 사용자

단독 사용자는 서버에 연결하지 않고 현재 기기 안에서만 메모를 저장합니다.

이 방식이 맞는 경우:

- 휴대폰이나 PC 한 대에서만 메모를 관리할 때
- 서버 설치, 계정, 토큰 같은 설정 없이 바로 쓰고 싶을 때
- 개인 메모를 외부 서버에 올리고 싶지 않을 때
- 테스트나 임시 기록 용도로 가볍게 쓰고 싶을 때

사용 가능한 기능:

- 일자별 메모
- 지식 메모
- 검색
- 태그
- 백링크와 연결 보기
- Markdown 작성과 미리보기
- 삭제 보관함
- 메모 단위 암호화 저장
- 백업과 복원(설치형/개발용 Web JSON, 모바일 DB)
- Markdown 가져오기와 내보내기
- 화면 설정
- 단축키 설정

제한되는 기능:

- 다른 기기와 자동 동기화
- 서버 백업
- 서버 기반 사용자 관리
- 서버 기반 LLM 분석 작업

단독 사용자는 주기적으로 백업 또는 Markdown 내보내기를 해두는 것이 좋습니다. 설치형/개발용 Web은 JSON 백업, 모바일 앱은 DB 백업을 사용합니다.

### 2. 서버 연결 사용자

서버 연결 사용자는 NowNote 서버에 접속해서 메모를 동기화합니다.

서버 연결 방식은 두 가지입니다.

- 개인 서버: 사용자가 Docker 컨테이너로 자기 서버를 직접 설치
- 공용 서버: 기존에 운영 중인 NowNote 서버에 사용자가 직접 가입해 접속

이 방식이 맞는 경우:

- 휴대폰, PC, 설치형 프로그램에서 같은 메모를 보고 싶을 때
- 서버에 백업본을 남기고 싶을 때
- 사용자 계정, 기기, 접속 상태를 관리해야 할 때
- 서버 기반 분석 기능을 사용하고 싶을 때
- 여러 기기에서 암호화 메모까지 같은 키로 열어보고 싶을 때

사용 가능한 기능:

- 단독 사용자 기능 전체
- 서버 동기화
- 서버 백업
- 서버 백업 검증
- 사용자 프로필
- 사용자별 시간대
- 사용자 그룹
- 활성 상태 관리
- 2단계 인증 사용 여부 관리
- 기기별 동기화 이력 확인
- 서버 운영 점검 화면
- 서버 기반 녹음 파일 저장
- 서버 기반 분석 작업

주의할 점:

- 설치형 프로그램과 모바일 앱은 서버 주소, 사용자 ID, 앱/설치형 접속 토큰으로 서버에 연결합니다.
- Web 프로그램은 서버 주소에서 사용자 ID와 비밀번호로 로그인합니다.
- 구형 개인 서버 API 토큰은 2.3 기본 사용 흐름에서는 필요 없으며, 기존 개인 서버 호환이 필요할 때만 고급 설정에서 사용합니다.
- 개인 서버 사용자는 `.env`의 토큰과 DB 비밀번호를 반드시 변경해야 합니다.
- 공용 서버 사용자는 Web에서 직접 가입하고, 로그인 후 앱/설치형 접속 토큰을 직접 발급합니다.
- 현재 1차 서버는 개인 Docker 서버 기준의 구형 개인 서버 API 토큰으로 시작할 수 있고, 공용 서버용 사용자 직접 가입, 앱/설치형 접속 토큰, Web 로그인, 비밀번호 재설정, 2단계 코드 검증도 함께 제공합니다.
- 공용 서버 오픈 전 `scripts/preflight.py --public-server` 실패 항목을 모두 해소해야 합니다. 현재 공용 서버 기준에서는 사용자 직접 가입, Web 로그인, 앱/설치형 접속 토큰, HTTPS/reverse proxy, 비밀번호 재설정 메일 설정을 모두 확인합니다.
- 서버 백업 검증은 백업 파일의 스키마, 체크섬, 필수 항목, 토큰 민감정보 노출 여부를 확인합니다.
- 서버에 연결해도 보관된 일자별 메모는 삭제하지 않고 비활성 백업 상태로 유지합니다.

## 플랫폼별 기준

### 모바일 앱

모바일 앱은 빠른 입력이 중심입니다.

설치 경로:

- 정식 배포 후: Google Play 스토어에서 NowNote 설치
- 내부 테스트 기간: 운영자가 제공한 Google Play 내부 테스트 링크로 설치
- 개발/검증용 직접 실행: `now_app/README.md`와 `now_app/docs/google_play_step_by_step_ko.md` 확인

서버를 연결하지 않아도 모바일 앱은 단독으로 사용할 수 있습니다.
서버 연결 설정을 비워두면 메모는 현재 휴대폰 안에 저장됩니다.
공용 서버에 접속하려면 먼저 Web에서 가입하고 앱/설치형 접속 토큰을 발급한 뒤, 앱 설정 화면의 서버 연결 항목에 서버 주소와 접속값을 입력합니다.

권장 사용 흐름:

- 홈에서 오늘 메모를 빠르게 작성
- 음성 또는 타이핑으로 일자별 메모 추가
- 필요한 경우 계층 메모 작성
- 서버 연결 사용자는 설정에서 서버 주소, 사용자 ID, 앱/설치형 접속 토큰 입력
- 구형 개인 서버 API 토큰은 기존 개인 서버 호환이 필요할 때만 고급 설정에서 입력
- 2단계 인증 코드는 저장하지 않고 연결 확인 때만 입력

모바일에서 중요한 기능:

- 일자별 메모
- 음성 메모
- 빠른 메모 입력
- 서버 동기화
- 연결 상태 확인
- 서버 녹음 업로드 상태 확인
- 서버 분석 작업 등록과 결과 확인

### Web 프로그램

Web 프로그램은 외부 PC에서 브라우저로 접속하는 서버 제공 화면입니다.
Web은 로컬 메모장이 아니라 서버에 공유된 내 문서만 조회하고 편집합니다.
서버를 직접 설치하지 않는 사용자는 공용 Web 프로그램 `https://nownote.sinsan.kr`에 접속합니다.
개인 서버를 운영하는 사용자는 `https://내도메인` 또는 `http://서버IP:8750`으로 접속합니다.
Web은 사용자 ID와 비밀번호로 로그인하며, 사용자가 앱/설치형 접속 토큰을 로그인 화면에 붙여넣는 방식으로 쓰지 않습니다.

개발자가 확인하는 `web/index.html`은 사용자용 Web 프로그램 주소가 아닙니다.
사용자용 Web 프로그램은 서버 주소 자체에서 열립니다.
`/app/`은 이전 안내와 호환하기 위한 보조 주소입니다.

권장 사용 흐름:

- 주제 / 분류 / 메모 구조로 지식 정리
- Markdown으로 긴 글 작성
- 검색, 태그, 백링크, 연결 보기로 메모 탐색
- 필요할 때 일자별 메모를 팝업처럼 열어 확인
- 단축키와 기능 표시 여부를 사용자 취향에 맞게 조정

Web에서 중요한 기능:

- 계층형 지식 메모
- 탭 기반 편집
- 본문 찾기
- Markdown 보기
- 빠른 전환
- 연결 보기
- 작업공간 저장과 지식 건강 점검
- 기능별 ON/OFF
- 단축키 변경
- Markdown 가져오기와 내보내기
- 서버 분석 작업 등록, 결과 요약 확인, 메모 본문에 추가

작업공간 기능은 현재 열린 탭, 선택 메모, 검색어, 그래프 필터, 속성 필터, 화면 상태를 저장합니다.
지식 건강 점검은 고립 메모, 오래된 메모, 과도하게 연결된 메모, 속성 누락 메모를 보여주고 현재 메모의 외부 링크를 안전하게 새 창으로 열 수 있게 합니다.

### 설치형 프로그램

Windows 설치형 프로그램은 집 PC에서 사용하는 PC용 NowNote입니다.
화면 디자인은 Web과 비슷하지만 내부 동작은 다릅니다.
설치형 프로그램은 PC 로컬 문서와 서버 공유 문서를 함께 다룹니다.
서버를 연결하지 않으면 PC 안에서 단독으로 사용할 수 있고, 서버를 연결하면 공유로 선택한 지식 메모와 일자별 메모를 동기화합니다.

Windows 설치형 프로그램은 `.exe` 설치 파일로 제공합니다.
설치와 실행 기준은 `desktop/README.md`에서 확인합니다.

## 서버 연결 설정

앱과 설치형 프로그램의 서버 연결에 필요한 값:

- 서버 주소
- 사용자 ID
- 앱/설치형 접속 토큰: 설치형 프로그램과 모바일 앱의 기본 서버 연결 수단
- 2단계 인증 코드: 2단계 인증을 사용하는 사용자가 연결 확인 때만 입력
- 구형 개인 서버 API 토큰: 기존 개인 서버 호환이 필요할 때만 고급 설정에서 사용
- 기기 ID: 기본 사용자는 직접 입력하지 않고 필요 시 고급 설정에서 확인

Web 프로그램은 서버 주소에서 사용자 ID와 비밀번호로 로그인합니다.
공용 서버에서 구형 개인 서버 API 토큰은 일반 사용자의 기본 연결값이 아닙니다.

### 서버를 직접 설치하지 않는 사용자

서버를 설치하지 않고 단독으로만 쓰는 경우 서버 연결 설정은 비워둡니다.

먼저 볼 경로:

```text
사용자 도움말: docs/HELP.md
모바일 앱 안내: now_app/README.md
설치형 프로그램 안내: desktop/README.md
공용 Web 프로그램: https://nownote.sinsan.kr
공용 개인정보처리방침: https://nownote.sinsan.kr/privacy
```

공용 서버에 접속만 하는 사용자는 Web에서 직접 가입하고 필요한 연결값을 직접 발급합니다.

```text
Web 주소: 예) https://nownote.sinsan.kr
Web 로그인: 사용자가 가입한 사용자 ID와 비밀번호
앱/설치형 서버 주소: 예) https://nownote.sinsan.kr
앱/설치형 사용자 ID: 사용자가 가입한 사용자 ID
앱/설치형 접속 토큰: Web 로그인 후 사용자가 발급한 연결 토큰
구형 개인 서버 API 토큰: 공용 서버에서는 입력하지 않음
2단계 인증 코드: 2단계 인증을 사용하는 경우 연결 확인 때만 6자리 코드 입력
기기 ID: 앱 또는 설치형 프로그램에서 자동 생성되며 필요 시 고급 설정에서 확인
```

입력 위치:

```text
모바일 앱: 설정 화면의 서버 연결 항목
설치형 프로그램: 화면 설정의 서버 연결 항목
Web 프로그램: https://nownote.sinsan.kr 또는 https://내도메인의 로그인 화면
```

사용자는 서버의 `.env` 파일을 직접 볼 필요가 없습니다.
비밀번호를 잊으면 Web 로그인 화면에서 등록 이메일로 재설정 코드를 받아 새 비밀번호를 설정합니다.

개인 Docker 서버 예시:

```text
Web 주소: http://서버주소:8750
Web 로그인: Web에서 가입했거나 관리자 화면에서 만든 사용자 ID와 Web 비밀번호
앱/설치형 서버 주소: http://서버주소:8750
구형 개인 서버 API 토큰: 기존 개인 서버가 NOW_API_TOKEN 보호를 쓰는 경우에만 고급 설정에서 입력
앱/설치형 접속 토큰: Web에서 직접 발급한 토큰
2단계 인증 코드: 필요한 경우 연결 확인 때만 6자리 코드 입력
사용자 ID: local_user 또는 Web에서 가입한 사용자 ID
기기 ID: 앱 또는 설치형 프로그램에서 자동 생성되며 필요 시 고급 설정에서 확인
```

개인 서버를 Linux 서버에 설치한 경우, 휴대폰이나 외부 PC에서는 `localhost`를 쓰지 않습니다.
서버의 도메인 또는 서버 IP를 사용합니다.

## 사용자 프로필

서버 연결 사용자는 서버에 사용자 정보가 생성됩니다.

사용자 정보:

- owner ID
- 이메일
- 표시 이름
- 시간대
- 사용자 그룹
- 2단계 인증 사용 여부
- 활성 상태
- 최근 접속 시간

앱과 설치형 프로그램은 자기 프로필의 이메일, 표시 이름, 시간대를 수정할 수 있습니다.
모바일 앱과 설치형 프로그램에서는 서버 설정 화면에서 사용자 ID를 입력한 뒤 사용자 프로필을 불러오거나 저장합니다.
Web 프로그램에서는 로그인한 사용자 기준으로 서버의 공유 문서를 사용합니다.
서버 제공 Web 로그인에서는 같은 사용자 그룹에 속한 다른 사용자가 서버에 공유한 지식 메모를 함께 조회할 수 있지만, 수정하거나 삭제할 수 없습니다.
그룹 공유 메모는 읽기 전용으로 표시되며 내 계정의 동기화 업로드 대상에 포함되지 않습니다. 설치형/모바일 토큰 동기화에는 표시하지 않습니다.

관리자는 서버 관리 화면에서 사용자 그룹, 2단계 인증 사용 여부, 활성 상태를 관리합니다.
공용 서버 운영자는 사용자를 대신해 계정을 만들 필요가 없고, 관리 화면에서 사용자 그룹, 활성 상태, 2단계 인증 사용 여부, 이상 접속 상태를 관리합니다.
비활성 사용자는 프로필 조회 외의 서버 동기화와 데이터 API 사용이 차단됩니다.
2단계 인증을 사용하는 사용자는 토큰 로그인 확인 때 6자리 인증 코드를 함께 입력해야 합니다.

## 백업과 가져오기

### 설치형 / 개발용 Web JSON 백업

JSON은 개발용 Web 화면 또는 설치형 NowNote 데이터를 백업하거나 복원할 때 사용합니다.
공용 Web 프로그램의 원본은 서버 공유 문서이므로 운영 백업은 서버 백업 절차를 우선합니다.

포함되는 내용:

- 일자별 메모
- 보관된 일자별 메모
- 지식 메모
- 삭제 보관함
- 화면 설정
- 열린 탭 설정

JSON 가져오기는 현재 데이터를 백업 파일 내용으로 교체하는 성격이 강합니다.

### 모바일 DB 백업

모바일 앱은 `.db` 파일로 전체 데이터를 백업하거나 복원합니다.

- 설정 > 데이터 관리 > 백업 내보내기에서 DB 백업 파일을 저장합니다.
- 백업 가져오기는 현재 모바일 데이터를 백업 파일 내용으로 교체합니다.
- 가져오기 후에는 앱을 다시 실행해야 합니다.

### Markdown 가져오기

Markdown 가져오기는 다른 곳에서 작성한 `.md`, `.markdown`, `.txt` 파일을 NowNote 지식 메모로 추가하는 기능입니다.

가져온 파일은 원본 파일과 연결하지 않습니다.

- 원본 파일은 그대로 둡니다.
- NowNote 내부에 새 메모를 만듭니다.
- NowNote에서 수정해도 원본 파일은 바뀌지 않습니다.
- NowNote에서 삭제해도 원본 파일은 삭제되지 않습니다.

### Markdown 내보내기

Markdown 내보내기는 사람이 읽을 수 있는 문서 형태로 지식 메모, 일자별 메모, 보관된 일자별 메모를 저장합니다.

내보내기 기준:

- 현재 화면 언어에 맞춰 한국어 또는 영어 형식으로 생성합니다.
- 지식 메모 제목에는 주제 / 분류 / 메모 단계가 표시됩니다.
- 일자별 메모와 보관된 일자별 메모도 함께 포함합니다.
- NowNote에서 내보낸 한국어 Markdown과 영어 Markdown은 모두 다시 가져올 수 있습니다.

## 암호화 저장

지식 메모는 필요할 때 메모 단위로 암호화할 수 있습니다.

암호화한 메모는 서버와 동기화되어도 암호문으로 저장됩니다. Web, 설치형 프로그램, 앱에서 같은 메모를 열려면 같은 키를 입력해야 합니다.

원칙:

- 로그인은 사용 권한 확인에 사용합니다.
- 실제 암호화 키는 서버나 브라우저 저장소에 저장하지 않습니다.
- 암호화된 메모는 서버 운영자도 원문을 읽을 수 없어야 합니다.
- 암호화 메모는 기본적으로 LLM 분석에서 제외합니다.
- 복호화는 임시로 내용을 여는 동작입니다.
- 암호화 해제는 메모를 다시 평문으로 저장하는 동작입니다.

키를 잊으면 NowNote가 내용을 복구할 수 없습니다.

## 추천 시작 방법

처음에는 단독 사용자로 시작해도 됩니다.

1. Web 또는 앱에서 메모 구조를 먼저 정합니다.
2. 일자별 메모와 지식 메모를 사용해 봅니다.
3. 여러 기기에서 같은 메모가 필요해지면 서버를 연결합니다.
4. 서버 연결 후 사용자 프로필과 시간대를 확인합니다.
5. 운영이 안정되면 백업과 분석 기능을 단계적으로 켭니다.
