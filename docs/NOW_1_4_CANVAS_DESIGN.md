# NowNote 1.4 Canvas와 시각적 사고 정리 설계서

기준일: 2026-06-01

이 문서는 NowNote 1.4 완료 여부를 판단하는 기준 문서입니다.

## 목표

그래프뷰가 자동으로 관계를 보여주는 화면이라면 Canvas는 사용자가 직접 생각을 배치하고 연결하는 작업 공간입니다.
Obsidian Canvas의 장점을 NowNote 방식으로 흡수하되, 1.4에서는 안정적인 1차 Canvas를 목표로 합니다.

## 포함 범위

- Canvas 보드 1차 화면
- 메모 카드 추가
- 텍스트 카드 추가
- 카드 위치 이동
- 카드 간 연결선 추가
- 연결 라벨
- 확대/축소 1차
- 전체 보기
- 선택 메모와 그래프 주변 메모를 Canvas 초안으로 만들기
- Canvas 저장 구조
- Canvas 카드에서 원본 메모 열기

## 제외 범위

- 무한 줌/팬 전체 구현
- 이미지/첨부/미디어 카드
- Canvas 서버 충돌 전용 UI
- JSON Canvas 가져오기/내보내기
- 모바일 Canvas 동등화
- 그림 그리기 기능

제외 사유:

- 사진/첨부/그림은 1.5의 첨부/미디어/빠른 기록 범위입니다.
- 서버 충돌 전용 UI와 JSON Canvas 호환은 Canvas 저장 구조가 안정된 뒤 확장합니다.

## 데이터 기준

Canvas 데이터는 `state.data.canvases`에 저장합니다.

- `id`
- `title`
- `cards`
- `edges`
- `zoom`
- `createdAt`
- `updatedAt`

카드는 다음 유형을 가집니다.

- `note`: 기존 지식 메모를 참조하는 카드
- `text`: Canvas 안에서만 쓰는 텍스트 카드

## 화면 기준

- 사용자는 Canvas 버튼으로 보드를 열 수 있습니다.
- 선택한 메모를 메모 카드로 추가할 수 있습니다.
- 텍스트 카드를 추가하고 내용을 수정할 수 있습니다.
- 카드를 선택하고 방향 버튼으로 위치를 조정할 수 있습니다.
- 선택한 카드 2개를 연결선으로 묶을 수 있습니다.
- 그래프 주변 메모를 Canvas 초안으로 만들 수 있습니다.
- 메모 카드를 눌러 원본 메모를 열 수 있습니다.

## 회귀 방지 기준

- 기존 지식 메모 제목/본문 저장 흐름은 변경하지 않습니다.
- Canvas는 삭제 보관함 메모를 자동으로 포함하지 않습니다.
- Canvas 카드가 참조하는 메모가 없어져도 Canvas 화면은 깨지지 않고 누락 카드로 표시합니다.
- Web은 서버 공유 문서 기준 데이터 안에서 Canvas를 다루고, 설치형은 PC 로컬 저장소 기준 데이터 안에서 Canvas를 다룹니다.
