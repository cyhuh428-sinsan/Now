# NowNote 1.9 작업공간과 운영형 지식 관리 설계서

이 문서는 NowNote 1.9 완료 여부를 판단하는 기준 문서입니다.

## 목표

NowNote 1.9의 목표는 2.0 LLM 지식화로 넘어가기 전에, 많은 지식 메모를 반복적으로 관리할 수 있는 작업 표면을 완성하는 것입니다.
Obsidian의 Workspaces, Outline, Word count, Tags view, Search, Web viewer, File explorer에서 NowNote에 필요한 부분만 선별해 흡수합니다.

## 범위

1. 작업공간 저장과 전환
   - 열린 지식 메모 탭, 고정 탭, 선택 메모를 저장합니다.
   - 검색어, 그래프 필터, 속성 보기 필터를 저장합니다.
   - 사이드바, 지식 목록 폭, 넓은 작성 공간 같은 화면 상태를 저장합니다.
   - 저장한 작업공간을 다시 적용할 수 있습니다.

2. 운영형 지식 건강 점검
   - 고립 메모를 확인합니다.
   - 오래된 메모를 확인합니다.
   - 과도하게 연결된 허브 메모를 확인합니다.
   - 속성 누락 메모를 확인합니다.
   - 정리 우선순위를 화면에서 바로 열 수 있습니다.

3. 목차와 통계 보강
   - 현재 메모의 제목 구조를 개요로 확인합니다.
   - 단어 수, 문자 수, 읽기 시간을 표시합니다.
   - 현재 메모의 외부 링크를 확인하고 안전하게 새 창으로 엽니다.

4. 파일 탐색형 운영 흐름
   - 기존 계층형 지식 목록을 작업공간 상태와 함께 복원합니다.
   - 검색, 태그, 속성 보기, 그래프 보기에서 같은 작업 상태를 공유합니다.

## 제외 범위

- 실시간 공동 편집
- 브라우저 안 iframe 기반 외부 사이트 렌더링
- 서버 기반 장기 LLM 작업 큐
- 메모 원본 파일 시스템 직접 편집

위 항목은 2.0 또는 이후 협업/서버 작업 단계에서 별도 판단합니다.
