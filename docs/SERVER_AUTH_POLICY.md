# NowNote 서버 인증 기준

이 문서는 1차 서버에서 개인 Docker 서버와 공용 NowNote 서버의 인증 기준을 구분하기 위한 운영 메모입니다.

## 2.3 인증 역할 기준

- Web은 사용자 ID와 비밀번호로 로그인하고, 서버가 발급한 짧은 Web 세션을 `X-Now-Web-Session` 헤더로 보냅니다.
- 설치형 프로그램과 모바일 앱은 서버 주소, 사용자 ID, 앱/설치형 접속 토큰을 기본 연결 값으로 사용하고, 이 값은 `X-Now-User-Token` 헤더로 검증합니다.
- 개인 서버 API 토큰인 `NOW_API_TOKEN`은 2.3 기본 사용자 흐름에서는 필요 없으며, 기존 개인 서버 호환 또는 운영자/관리자 API 보호가 필요할 때만 사용합니다.
- 구형 호환 기준으로 모바일 앱과 설치형 프로그램은 서버 주소, API 토큰 또는 사용자별 접속 토큰, 사용자 ID, 기기 ID를 저장해 서버에 연결할 수 있습니다.
- 2단계 인증 코드는 저장하지 않고 연결 확인, 토큰 확인, Web 로그인 때만 입력합니다.

## 현재 구현

- 서버 API 보호는 `.env`의 `NOW_API_TOKEN`을 구형 개인 서버 호환/운영 보호용으로 유지합니다.
- `NOW_API_TOKEN`은 개인 서버 기본 모드와 운영자/관리자 API 보호용 값이며 일반 사용자의 Web 로그인 값이 아닙니다.
- Web 프로그램은 서버 주소에서 사용자 ID와 비밀번호로 로그인하고, 서버가 발급한 짧은 Web 세션으로 서버 공유 문서에 접근합니다.
- 2단계 인증 사용자는 연결 테스트, 토큰 확인 또는 Web 로그인 때 6자리 인증 코드를 함께 입력합니다. 2단계 코드는 저장 대상이 아니라 확인 요청에만 사용합니다.
- 사용자 ID(`owner_id`)별 프로필, 시간대, 그룹, 2단계 인증 사용 여부, 활성 상태는 서버에 저장됩니다.
- Web에서 사용자가 직접 가입하고 Web 로그인 비밀번호를 설정할 수 있습니다.
- Web에서 사용자가 앱/설치형 연결 토큰을 발급하고 다시 확인할 수 있습니다.
- 등록 이메일 기반 비밀번호 재설정 API가 준비되어 있으며, 공용 서버에서는 SMTP 설정이 필요합니다.
- 관리자 화면과 API에서는 운영자 계정 선등록, 테스트 계정, 장애 대응용으로 Web 로그인 비밀번호와 사용자별 접속 토큰을 관리할 수 있습니다.
- 관리자 발급 사용자별 접속 토큰은 원문을 저장하지 않고 해시와 발급 시각만 저장합니다.
- Web에서 발급한 기기별 연결 토큰은 사용자가 다시 볼 수 있도록 서버 DB에 보관하지만, 관리자 export와 백업 JSON에는 포함하지 않습니다.
- 앱/설치형 토큰 확인 API `/api/v1/auth/token-login`이 준비되어 있으며, 성공 응답은 사용자 ID, 그룹, 활성 상태, 토큰 종류, 마지막 사용 시각을 반환합니다.
- Web 로그인 API `/api/v1/auth/web-login`, Web 세션 확인 API `/api/v1/auth/web-session`, Web 로그아웃 API `/api/v1/auth/web-logout`이 준비되어 있습니다.
- 사용자 토큰 확인 화면 `/auth/token`과 앱/설치형 토큰 확인 API `/api/v1/auth/token-login`이 준비되어 있습니다.
- 비활성 사용자는 프로필 조회 외의 동기화, 메모, 녹음, 분석 API가 차단됩니다.
- `NOW_USER_TOKEN_REQUIRED=false`인 개인 서버 기본 모드에서는 앱/설치형 데이터 API가 구형 개인 서버 API 토큰인 `NOW_API_TOKEN`을 확인합니다.
- `NOW_USER_TOKEN_REQUIRED=true`이면 앱/설치형 데이터 API는 `X-Now-User-Token` 헤더를 사용자 ID와 대조합니다.
- Web 프로그램은 `X-Now-Web-Session` 헤더로 로그인 세션을 검증합니다.
- 사용자별 기기 목록 조회와 기기 활성/비활성 변경 API가 준비되어 있습니다.
- 2단계 인증 사용자는 토큰 로그인 때 6자리 추가 코드를 검증합니다.
- 일자별 메모는 서버 공유 대상으로 저장하고, 지식 메모는 사용자가 공유로 선택한 항목만 서버에 저장합니다.
- 지식 메모는 주제/분류/메모 3단계 계층 구조를 서버에 보존하며, 중간 단계 없이 하위 메모만 단독 저장하는 요청은 거부합니다.

## 개인 Docker 서버

개인 서버는 서버 소유자와 사용자가 사실상 같은 운영 모델입니다.

- `NOW_API_TOKEN`을 긴 랜덤 값으로 바꿉니다.
- `NOW_USER_TOKEN_REQUIRED=false`로 시작할 수 있습니다.
- DB 비밀번호를 첫 실행 전에 확정합니다.
- 서버는 외부에 공개하지 않거나, 필요한 사람에게만 계정과 사용자별 접속 토큰을 발급합니다.
- 사용자 ID는 `local_user`로 시작해도 되고, 여러 기기를 구분하려면 운영자가 정한 owner ID를 사용할 수 있습니다.

이 모델에서는 1차 인증 구조로 실제 사용이 가능합니다.

## 공용 NowNote 서버

공용 서버는 서로 다른 사용자가 같은 서버에 접속하는 운영 모델입니다.

현재의 단일 `NOW_API_TOKEN`만으로는 공용 서버를 정식 오픈하기에 부족합니다. 사용자 직접 가입, 앱/설치형 접속 토큰, Web 로그인, Web 세션, 이메일 비밀번호 재설정, 2단계 코드 검증, 데이터 API 검증 단계까지 준비되어 있으며, 공용 서버 오픈 전에는 아래 기능을 추가로 확정해야 합니다.

- HTTPS, reverse proxy, 운영 알림
- 비밀번호 재설정용 SMTP 설정
- 실제 설정 절차와 Nginx/Caddy 예시는 `server/PUBLIC_SERVER.md`와 `server/reverse_proxy` 폴더를 기준으로 확인합니다.

관리 화면의 사용자 그룹, 활성 상태, 2단계 인증 사용 여부, 사용자 토큰 확인 화면/API, 2단계 코드 검증 절차, 사용자별 기기 조회/해제 API, 사용자별 데이터 격리 자동 검증은 공용 서버 운영을 위한 모니터링 기능입니다. 공개 운영 절차가 완성되기 전까지는 제한된 테스트 서버로만 사용합니다.

## 점검 명령

개인 Docker 서버 배포 전:

```bash
python3 scripts/preflight.py
```

공용 서버 오픈 전:

```bash
python3 scripts/preflight.py --public-server
```

사용자별 토큰 필수 모드에서 서버 동작을 점검할 때:

```bash
python3 scripts/smoke_test.py --base-url http://localhost:8750 --token 긴-랜덤-토큰 --issue-local-user-token
```

`--public-server`는 `NOW_USER_TOKEN_REQUIRED=true`, `NOW_PUBLIC_BASE_URL=https://도메인`, `NOW_BEHIND_REVERSE_PROXY=true`가 설정되지 않으면 의도적으로 실패합니다. 이 실패는 공용 서버에 필요한 사용자별 접속 토큰 강제와 공개 운영 절차가 아직 남아 있음을 알려주는 안전장치입니다.
