# NowNote 2.2 그룹 메신저 설계서

## 목적

2.2의 목표는 같은 사용자 그룹 안에서 짧은 운영 대화를 Web 화면으로 주고받을 수 있게 하는 것이다.

이 기능은 지식 메모 공유와 별개의 대화 흐름이며, 설치형 프로그램과 모바일 앱에는 2.2 범위에서 포함하지 않는다.

## 적용 범위

| 구분 | 내용 |
| --- | --- |
| 제공 위치 | 서버 hosted Web |
| 사용자 범위 | 같은 `group_name`을 가진 활성 사용자 |
| 권한 기준 | Web 로그인 세션 필요 |
| 저장 단위 | 그룹 메시지 |
| 제외 | 개인 DM, 파일 첨부, 읽음 확인, 알림 push, 모바일/설치형 메신저 |

## 사용자 흐름

1. 사용자가 hosted Web에 로그인한다.
2. 관리 메뉴의 `그룹 메신저`를 연다.
3. 같은 그룹의 최근 메시지를 시간순으로 조회한다.
4. 메시지를 작성해 전송한다.
5. 전송된 메시지는 같은 그룹 사용자에게만 조회된다.

## 운영 기준

- 그룹 메신저는 Web 세션이 있는 사용자만 접근한다.
- 다른 그룹의 메시지는 API에서 조회하거나 작성할 수 없다.
- 메시지는 작성자 본문과 생성 시각만 저장한다.
- 계정 삭제 시 해당 사용자가 보낸 메시지도 삭제한다.
- 운영자는 DB/CLI가 아니라 향후 운영 화면/API를 통해 점검할 수 있어야 한다.

## 데이터 구조

`group_messages`

| 필드 | 설명 |
| --- | --- |
| `id` | 서버 메시지 ID |
| `group_name` | 메시지가 속한 사용자 그룹 |
| `sender_owner_id` | 작성자 사용자 ID |
| `body` | 메시지 본문 |
| `created_at` | 작성 시각 |

## 완료 기준

- hosted Web에 `그룹 메신저` 메뉴가 표시된다.
- Web 세션 없이는 그룹 메시지 API를 사용할 수 없다.
- 같은 그룹 메시지만 조회된다.
- 메시지를 전송하면 목록에 반영된다.
- 서버 capability에 `group_messenger`가 표시된다.
- 기존 지식 메모 공유, 동기화, 설치형/모바일 동작은 변경되지 않는다.
