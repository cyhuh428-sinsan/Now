# NowNote Web/설치형 실제 실행 점검서

이 문서는 NowNote Web을 PC 설치형 프로그램처럼 사용하는 1차 기준 점검서입니다.
1차 설치형 포장 방식은 Windows `.exe` 설치 파일을 기준으로 합니다.
PWA 설치는 브라우저 기반 보조/검증 흐름으로 유지합니다.

## 점검 전 준비

- 최신 소스 반영: 저장소 루트에서 `git pull origin main`
- Web 폴더에서 임시 서버 실행: `python -m http.server 8761 --bind 127.0.0.1`
- 브라우저 주소: `http://127.0.0.1:8761/index.html`
- 서버 연결 점검이 필요하면 NowNote 서버를 먼저 `8750` 포트로 실행
- 직접 파일로 열면 서비스 워커와 PWA 설치가 제한될 수 있으므로 설치형 점검은 `127.0.0.1` 주소에서 진행

## 1. 기본 실행

- [ ] Web 화면이 열린다.
- [ ] 앱 이름이 NowNote로 표시된다.
- [ ] 지식 메모 화면이 첫 화면으로 보인다.
- [ ] 왼쪽 레일과 사이드바가 보인다.
- [ ] 화면 설정을 열고 닫을 수 있다.

## 2. 지식 메모

- [ ] 주제를 추가한다.
- [ ] 주제 아래 분류를 추가한다.
- [ ] 분류 아래 메모를 추가한다.
- [ ] 3단계 메모 아래에는 더 이상 하위 메모를 추가할 수 없다.
- [ ] 메모 제목과 본문을 수정하고 다시 선택해도 유지된다.
- [ ] 열린 메모 탭이 생기고 탭 전환이 된다.
- [ ] 삭제 보관함으로 이동하고 복원 또는 영구 삭제를 확인한다.

## 3. 일자별 메모

- [ ] 레일 또는 오늘 메모 버튼으로 일자별 메모 팝업을 연다.
- [ ] 오늘 날짜에 메모를 입력한다.
- [ ] 같은 날짜에 다시 입력하면 같은 메모장에 이어서 저장된다.
- [ ] 다른 날짜를 선택해 별도 메모를 확인한다.
- [ ] 보관 기능은 메모를 삭제하지 않고 비활성 보관 상태로 유지한다.

## 4. Markdown / JSON

자동 점검 가능 명령: `node scripts/check_import_export.mjs`

- [ ] Markdown 내보내기를 실행해 `.md` 파일이 내려받아진다.
- [ ] `.md`, `.markdown`, `.txt` 파일을 Markdown 가져오기로 가져온다.
- [ ] 일반 Markdown 파일은 새 주제로 추가된다.
- [ ] NowNote Markdown 내보내기 파일은 주제/분류/메모와 일자별 메모를 복원한다.
- [ ] JSON 내보내기를 실행해 `.json` 백업 파일이 내려받아진다.
- [ ] JSON 가져오기는 현재 상태를 먼저 자동 백업한 뒤 백업 파일 내용으로 교체한다.

## 5. 검색과 편집 편의

- [ ] 검색 팝업에서 제목과 본문을 검색한다.
- [ ] `path:`, `file:`, `tag:`, `line:` 접두어 검색을 확인한다.
- [ ] 본문 찾기에서 현재/전체 개수와 다음/이전 이동을 확인한다.
- [ ] Markdown 보기, 개요, 시간 넣기, 링크 복사 기능을 확인한다.
- [ ] 단축키 설정 화면에서 단축키 목록을 확인한다.

## 5-1. 관계 탐색과 그래프뷰

자동 점검 가능 명령: `node scripts/check_graph_view.mjs`

- [ ] 연결 보기를 열면 전체 그래프의 노드와 연결 목록이 표시된다.
- [ ] 선택 메모 중심 로컬 그래프와 깊이 1-3 전환을 확인한다.
- [ ] 검색어, 태그, 그룹 필터를 바꿔 그래프 범위가 달라지는지 확인한다.
- [ ] 고립 메모와 허브 메모 목록을 확인한다.
- [ ] 현재 메모의 나가는 링크와 백링크를 확인한다.
- [ ] 연결 후보를 눌러 본문에 `[[메모 제목]]` 링크가 반영되는지 확인한다.
- [ ] 그래프 필터 북마크를 저장하고 다시 적용한다.

## 5-2. 속성 기반 지식 관리

- [ ] 선택한 지식 메모에서 상태, 중요도, 유형, 출처, 작성자, 마감일, 프로젝트 속성을 수정한다.
- [ ] 속성 보기를 열어 메모가 속성 목록에 표시되는지 확인한다.
- [ ] 검색어, 상태, 중요도 필터를 적용한다.
- [ ] 상태, 중요도, 유형, 프로젝트, 공유 상태 그룹 전환을 확인한다.
- [ ] 속성 필터를 저장하고 다시 적용한다.
- [ ] 누락 속성 목록에서 메모를 열 수 있는지 확인한다.
- [ ] 템플릿으로 새 메모를 만들면 제목, 본문 섹션, 속성이 함께 생성되는지 확인한다.

## 5-3. Canvas와 시각적 사고 정리

- [ ] Canvas를 열고 선택 메모를 메모 카드로 추가한다.
- [ ] 텍스트 카드를 추가하고 내용을 수정한다.
- [ ] 카드 2개를 선택해 선으로 연결한다.
- [ ] 선택한 카드를 방향 버튼으로 이동한다.
- [ ] 확대/축소와 전체 보기를 확인한다.
- [ ] 그래프 초안 버튼으로 주변 메모 카드 묶음을 만든다.
- [ ] 메모 카드의 열기 버튼으로 원본 메모를 연다.

## 5-4. 첨부/미디어/빠른 기록

- [ ] 빠른 기록을 열고 제목 없이 내용을 저장한다.
- [ ] 핀, 색상, 라벨을 지정한 카드가 목록에 표시되는지 확인한다.
- [ ] 체크리스트 모드로 여러 줄을 저장한다.
- [ ] 시간 리마인더가 카드에 표시되는지 확인한다.
- [ ] 첨부 파일 선택 후 파일 이름, 유형, 용량이 카드에 표시되는지 확인한다.
- [ ] 간단한 그림을 그리고 카드에 저장한다.
- [ ] 보관한 카드는 활성 목록에서 사라지고 보관함 필터에 표시된다.

## 5-5. 작성 보조와 명령 체계

- [ ] 명령 팔레트를 열고 템플릿, 고유 메모, 랜덤 메모 명령이 검색되는지 확인한다.
- [ ] 프로젝트/회의/자료 템플릿으로 메모를 생성하고 속성이 함께 들어가는지 확인한다.
- [ ] 본문 한 줄에 `/time`, `/check`, `/template meeting`을 입력하고 Enter로 실행되는지 확인한다.
- [ ] `## 제목` 섹션이 있는 메모에서 나누기 명령을 실행하면 하위 메모가 생기는지 확인한다.
- [ ] 하위 메모가 있는 메모에서 병합 명령을 실행하면 원본 하위 메모를 삭제하지 않고 본문에 내용이 추가되는지 확인한다.

## 5-6. 복구/가져오기/마이그레이션

- [ ] 화면 설정에서 수동 스냅샷을 만들고 목록에 표시되는지 확인한다.
- [ ] Markdown 가져오기 전 자동 스냅샷이 남는지 확인한다.
- [ ] frontmatter가 있는 Markdown을 가져오면 상태, 중요도, 유형, 프로젝트 속성이 반영되는지 확인한다.
- [ ] Obsidian `[[문서|별칭]]`, `![[첨부]]` 표기가 보정되고 가져오기 진단에 표시되는지 확인한다.
- [ ] 선택한 스냅샷 복구를 실행하면 현재 상태가 먼저 스냅샷으로 남고 선택 상태로 복구되는지 확인한다.

## 5-7. 출판/발표/공개 지식 묶음

- [ ] 화면 설정에서 공개 묶음 제목, 설명, permalink를 저장할 수 있다.
- [ ] 공개 제외 메모와 암호화 메모가 공개 묶음에서 자동 제외되는지 확인한다.
- [ ] 민감정보 후보가 공개 전 목록에 표시되는지 확인한다.
- [ ] HTML 내보내기와 발표 HTML 내보내기를 실행한다.

## 5-8. 작업공간과 운영형 지식 관리

- [ ] 화면 설정에서 작업공간 이름을 입력하고 현재 탭, 검색, 그래프, 속성 필터 상태를 저장한다.
- [ ] 저장된 작업공간을 선택해 다시 열면 선택 메모와 화면 상태가 복원된다.
- [ ] 지식 건강 점검에서 고립 메모, 오래된 메모, 과도 연결 메모, 속성 누락 메모 후보를 확인한다.
- [ ] 지식 건강 점검 항목을 눌러 해당 메모로 이동한다.
- [ ] 현재 메모에 외부 URL이 있으면 작업공간 패널에서 링크 목록으로 표시되는지 확인한다.
- [ ] 포함 문서 체크 목록에서 공유 안 함, 암호화 메모, 공개 제외 속성, 비공개 태그 문서는 선택할 수 없다.
- [ ] 공개 전 민감정보 후보가 화면에 표시된다.
- [ ] HTML 내보내기 결과에 선택한 공개 가능 메모만 포함된다.
- [ ] 발표 HTML 내보내기 결과가 생성되고 `---` 구분선 기준으로 슬라이드가 나뉜다.

## 6. PWA 보조 설치 점검

- [ ] 브라우저 주소가 `http://127.0.0.1` 또는 HTTPS인지 확인한다.
- [ ] 브라우저의 앱 설치 또는 바로가기 설치 메뉴가 표시된다.
- [ ] 설치 후 독립 창으로 NowNote가 열린다.
- [ ] 독립 창에서 지식 메모와 일자별 메모가 기존 브라우저와 같은 데이터를 읽는다.
- [ ] 오프라인 또는 새로고침 후에도 기본 화면이 열린다.

## 7. 서버 연결

- [ ] 화면 설정에서 단독 사용과 서버 연결 모드를 전환할 수 있다.
- [ ] 서버 주소, 사용자 ID, 앱/설치형 접속 토큰, 2단계 인증 코드를 입력할 수 있고 구형 개인 서버 API 토큰과 기기 ID는 고급 설정에서만 보인다.
- [ ] 연결 테스트가 성공한다.
- [ ] 서버 capability와 공용 서버 준비 상태가 표시된다.
- [ ] 서버로 동기화와 전체 다시 동기화를 실행할 수 있다.
- [ ] 서버 `/admin` 또는 `/monitor` 화면에서 동기화 결과를 확인한다.

## 실패 시 확인 기준

- `Failed to fetch`가 나오면 서버 주소를 `http://127.0.0.1:8750`으로 바꿔 확인한다.
- 터미널의 `curl http://localhost:8750/health`는 성공하는데 브라우저만 실패하면 서버 컨테이너가 최신 CORS 설정으로 재배포됐는지 확인한다.
- PWA 설치 메뉴가 보이지 않으면 파일 직접 열기 여부, HTTPS 또는 `127.0.0.1` 실행 여부, 서비스 워커 등록 여부를 확인한다.
- 공용 서버는 HTTPS와 reverse proxy 설정이 끝난 뒤 점검한다.

## 완료 기록

실제 점검이 끝나면 `docs/PHASE1_RELEASE_CHECKLIST.md`의 Web/설치형 점검 항목을 실제 결과에 맞춰 갱신합니다.
