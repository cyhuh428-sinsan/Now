# NowNote Web

NowNote Web은 외부 PC에서 브라우저로 접속하는 서버 제공 Web 프로그램의 원본 화면입니다.
실제 사용자용 Web 프로그램은 서버 주소 자체에서 열립니다.
공용 서버를 쓰는 사용자는 `https://nownote.sinsan.kr`, 개인 서버를 쓰는 사용자는 `https://내도메인` 또는 `http://서버IP:8750`으로 접속합니다.
`/app/`은 이전 안내와 호환하기 위한 보조 주소입니다.

`web/index.html`은 개발/검증용 원본 파일입니다.
사용자에게 안내할 Web 주소로 쓰지 않습니다.

Web은 로컬 메모장의 원본이 아닙니다. 서버에 부속된 화면으로 동작하며, 사용자가 직접 가입하고 사용자 ID와 비밀번호로 로그인한 뒤 서버에 공유된 내 문서만 조회/편집합니다. 일자별 메모는 기본 공유 대상이고, 지식 메모는 공유 상태인 항목만 서버에 저장됩니다. 계층 메모는 주제/분류/메모 3단계 구조를 서버에 함께 저장하므로 중간 단계가 없는 손자 메모만 단독으로 저장하지 않습니다.

단독 사용자와 서버 연결 사용자의 차이는 [NowNote 도움말](../docs/HELP.md)을 기준으로 정리합니다.
영어 문서는 [NowNote Help](../docs/HELP.en.md)에서 확인할 수 있습니다.
Web/설치형 화면 언어는 한국어, 영어, 중국어, 일본어, 베트남어, 아랍어를 선택할 수 있습니다. 1.0에서는 한국어와 영어를 기준 언어로 두고, 추가 언어는 핵심 화면 번역과 영어 fallback으로 제공합니다.
Web 화면에서는 `help.html`로 같은 내용을 사용자용 화면에서 확인할 수 있습니다.
실제 실행 점검은 `runtime_checklist_ko.md`를 기준으로 확인합니다.

## 1차 범위

- 일자별 메모: 날짜마다 메모장 하나만 두고 계속 추가/편집
- 일자별 보관함: 오래된 날짜 메모를 활성 목록에서 빼고 보관 상태로 유지
- 지식 메모: 주제/분류/메모 3단계 구조
- 지식 목록: 주제 중심으로 접고 펼치는 구조
- 순서 변경: 같은 단계 안에서 지식 메모를 위/아래로 이동
- 열린 메모 탭: 선택한 지식 메모를 탭으로 유지하고 빠르게 전환, 탭 고정, 다른 탭/전체 탭 닫기, 닫은 탭 다시 열기, 열린 탭 제한 안에서 고정 탭 우선 보존
- 지식 편집: 메모 경로 표시, 기본 Markdown 미리보기
- Markdown 작성: 본문에서 제목/굵게/기울임/링크/인용/구분선/체크리스트/번호 목록/코드블록 빠른 입력, 들여쓰기, 일반/번호 목록과 인용 이어쓰기, 미리보기
- 클릭 경로: 편집 화면의 주제/분류/메모 경로를 눌러 바로 이동
- 본문 찾기: 선택한 지식 메모 안에서 단어를 검색하고 `현재/전체` 개수 표시, Enter/Shift+Enter 또는 위/아래 버튼으로 다음/이전 이동
- 개요: 현재 메모의 Markdown 제목 1-6단계를 목차와 미리보기에 표시하고, 기존 제목 줄도 단축키로 정리 가능
- 화면 설정: 밝은/어두운/시스템 테마, 강조 색상, 작성 공간 폭 조정
- 편집 설정: 글자 크기, 줄 간격, 백링크 표시 여부, 보조 목록 표시 여부, 화면 설정 초기화
- 언어팩: 한국어를 기본 언어로 유지하고, 설정 화면에서 한국어/English/中文/日本語/Tiếng Việt/العربية를 선택할 수 있는 언어팩 기반 제공. 추가 언어의 미번역 세부 문구는 영어로 표시
- 암호화 저장: 지식 메모 단위로 암호화, 복호화, 잠금, 암호화 해제 지원. 키는 서버나 브라우저 저장소에 저장하지 않음
- 아이콘 레일: 일일 메모, 검색, 화면 설정, 왼쪽 목록 접기/펼치기
- 작성 공간 조정: 지식 목록과 메모 작성 영역 사이의 손잡이를 드래그해서 폭 조정
- 빠른 전환: 최근 수정순 제목/경로 검색으로 지식 메모 바로 이동, 전체 후보와 표시 개수 구분, Enter/방향키 이동과 포커스 표시
- 검색 팝업: 레일의 검색 버튼으로 일자별 메모와 지식 메모를 별도 창에서 검색, 결과 개수와 검색어 하이라이트 표시, 범위/정렬 선택, Enter/방향키 이동
- 검색 접두어: `path:`, `file:`, `tag:`, `line:`으로 경로/제목/태그/내용 검색
- 1.2 관계 탐색과 그래프뷰: 전체 그래프, 선택 메모 중심 로컬 그래프, 깊이 1-3, 검색/태그/그룹 필터, 고립 메모, 허브 메모, 연결 후보, 그래프 필터 북마크를 제공
- 1.3 속성 기반 지식 관리: 상태, 중요도, 유형, 출처, 작성자, 마감일, 프로젝트 속성 편집, 속성 목록, 저장 필터, 누락 속성 점검, 템플릿 메모 생성을 제공
- 1.4 Canvas와 시각적 사고 정리: 메모 카드, 텍스트 카드, 카드 연결선, 확대/축소, 전체 보기, 그래프 주변 메모 Canvas 초안을 제공
- 1.5 첨부/미디어/빠른 기록: 제목 없는 기록 카드, 핀, 색상, 체크리스트, 시간 리마인더, 라벨, 보관함, 첨부 메타, 간단한 그림 저장을 제공
- 1.6 작성 보조와 명령 체계: 명령 팔레트, Slash command, 템플릿 메모, 고유 메모, 랜덤 메모, 합치기/나누기 1차를 제공
- 1.7 복구/가져오기/마이그레이션: 화면 기반 스냅샷, 가져오기 전 자동 스냅샷, 스냅샷 복구, frontmatter 속성 매핑, Obsidian 링크/첨부 보정 진단을 제공
- 1.8 출판/발표/공개 지식 묶음: 공개 묶음 저장, 공개 제외/암호화 메모 자동 제외, 민감정보 후보 점검, HTML/발표 HTML 내보내기를 제공
- 백링크/연결 보기: 본문에 `[[메모 제목]]`을 적으면 메모 연결을 표시하고 같은 메모의 중복 연결은 한 번만 표시
- 위키 링크: Markdown 미리보기에서 `[[메모 제목]]`을 클릭해 이동하거나 새 메모 생성
- 링크 복사: 선택한 메모의 `[[메모 제목]]` 링크를 바로 복사
- 메모 정보: 글자 수, 줄 수, 링크 수, 태그 수, 최근 수정 시간 표시
- 시간 넣기: 지식 메모 본문 커서 위치에 현재 시각을 바로 추가
- 태그: 본문에 `#태그`를 적으면 자동 인식하고 태그 전용 검색에 활용
- 즐겨찾기: 중요한 지식 메모를 표시
- 사이드바 탐색: 즐겨찾기와 태그 모음에서 바로 이동
- 최근 수정: 최근에 편집한 지식 메모를 사이드바에서 바로 열기
- 삭제 보관함: 지식 메모를 즉시 영구 삭제하지 않고 복원 가능한 상태로 보관, 삭제 성공 시 열린 탭 정리, 개수 표시, 개별/선택/전체 영구 삭제 지원
- 가져오기/내보내기: JSON 백업 가져오기/내보내기, Markdown 내보내기, `.md`/`.txt` 파일을 새 주제로 가져오기, NowNote Markdown 내보내기 파일의 주제/분류/메모와 일자별 메모 복원, 여러 Markdown 파일 동시 가져오기 지원
- 목록 제어: 지식 목록 전체 펼치기/접기
- 팝업 토글: 일일 메모, 검색, 빠른 전환, 연결 보기, 화면 설정, 삭제 보관함은 같은 버튼으로 열고 닫기
- 단축키: `Ctrl+N` 새 주제, `Ctrl+Shift+N` 선택 항목 아래에 추가, `Ctrl+S` 저장 상태 확인, `Ctrl+;` 시간 넣기, 본문에서 `Tab`/`Shift+Tab` 들여쓰기, 본문에서 `Ctrl+1/2/3` 제목, 본문에서 `Ctrl+B` 굵게, 본문에서 `Ctrl+I` 기울임, 본문에서 `Ctrl+Shift+L` 링크, 본문에서 `Ctrl+Shift+Q` 인용, 본문에서 `Ctrl+Shift+H` 구분선, 본문에서 `Ctrl+Shift+C` 체크리스트, 본문에서 `Ctrl+Shift+7` 번호 목록, 본문에서 `Ctrl+Shift+K` 코드블록, `Ctrl+F` 검색, `Ctrl+Shift+F` 본문 찾기, `Ctrl+K`/`Ctrl+O` 빠른 전환, `Ctrl+D` 일자별 메모, `Ctrl+G` 연결 보기, `Ctrl+W` 현재 탭 닫기, `Ctrl+Shift+T` 닫은 탭 다시 열기, `Ctrl+Shift+W` 다른 탭 닫기, `Ctrl+Shift+P` 현재 탭 고정, `Ctrl+PageUp`/`Ctrl+PageDown` 열린 탭 전환, `Ctrl+Alt+↑`/`Ctrl+Alt+↓` 같은 단계 순서 이동, `Ctrl+,` 화면 설정, `Esc` 팝업 닫기
- 검색: 일자별 메모와 지식 메모 통합 검색, 결과 개수와 검색어 하이라이트 표시, 검색 닫기, Enter/방향키 이동
- 저장 기준: 서버 제공 Web에서는 서버 공유 문서가 원본이며, 개발용 파일 실행에서만 브라우저 localStorage를 사용
- 설치형 저장 기준: 설치형 프로그램은 Electron 로컬 JSON 저장소를 우선 사용하고, 첫 실행 때 기존 localStorage 데이터를 1회 복사
- 공유 기준: 설치형에서 새로 만든 지식 메모는 기본 PC 로컬 문서이며, 공유 상태인 지식 메모와 일자별 메모만 서버 동기화 대상으로 처리
- 동기화 보호: 자동 동기화 ON/OFF, 충돌 문서 목록, 암호화 메모 분석 제외, 삭제 보관함 로컬 보관 기준을 유지
- 내보내기/가져오기: JSON 파일 기반 백업, 화면 설정과 열린 탭 설정 포함, 가져오기 전 파일명/시각/요약 확인과 현재 상태 자동 백업
- 가져오기 보정: 지식 메모를 불러올 때 실제 위치 기준으로 상위 관계와 단계를 정리
- Markdown 내보내기: 사람이 읽을 수 있는 지식 메모/일자별 메모/보관 메모 문서와 요약 생성, 지식 메모 제목에 주제/분류/메모 단계 표시, 현재 화면 언어 기준 내보내기 지원. 1.0 기준 언어는 한국어/영어이며 추가 언어는 영어 fallback 문구를 함께 사용
- 서버 분석: 선택한 지식 메모를 서버 분석 큐에 등록하고 최근 분석 작업 상태/결과 요약을 확인한 뒤 메모 본문에 추가
- 암호화 정책: 로그인은 사용 권한 확인에 사용하고, 실제 암호화 키는 사용자가 직접 입력하며 서버/브라우저 저장소에 저장하지 않음. 암호화 메모는 기본적으로 LLM 분석에서 제외
- 화면 알림: 단순 안내와 오류는 브라우저 기본 경고창 대신 앱 안 토스트로 표시

## 방향

처음 방향은 Obsidian처럼 가볍게 기록하고 빠르게 찾는 쪽을 우선합니다.
Trilium처럼 강한 관리형 구조는 필요한 기능만 천천히 가져옵니다.

## 실행과 배포 기준

사용자용 Web 프로그램 주소:

```text
공용 서버: https://nownote.sinsan.kr
개인 서버: https://내도메인
개인 서버 직접 접속: http://서버IP:8750
```

서버 Docker 이미지에는 `index.html`, `app.js`, `styles.css`, `help.html`, `manifest.webmanifest`, `sw.js`, `icons/`가 포함되고, FastAPI가 루트 주소에서 정적 파일로 제공합니다.
브라우저가 서버 루트 또는 `/app/`에서 실행될 때는 Web 로그인 화면이 열리고 서버 주소가 현재 도메인으로 자동 설정됩니다.

개발자가 로컬 화면을 빠르게 확인할 때만 아래 파일을 직접 열 수 있습니다.

```text
D:\Project\Now\web\index.html
```

Web 원본 화면의 핵심 요소가 빠지지 않았는지는 아래 명령으로 정적 점검합니다.

```bash
python scripts/verify_web_surface.py
```

Markdown 가져오기/내보내기와 JSON 백업/복원은 Chrome 또는 Edge가 설치된 환경에서 아래 명령으로 실제 브라우저 흐름을 확인합니다.

```bash
node scripts/check_import_export.mjs
```

설치형 1.1의 공유/충돌/암호화/삭제 보관함 정책은 아래 명령으로 실제 브라우저 함수 결과를 확인합니다.

```bash
node scripts/check_desktop_client_policies.mjs
```

1.2 그래프뷰의 전체/로컬 그래프, 고립/허브 메모, 연결 후보, 북마크 저장은 아래 명령으로 실제 브라우저 흐름을 확인합니다.

```bash
node scripts/check_graph_view.mjs
```

브라우저 실행 파일을 자동으로 찾지 못하거나 headless 브라우저가 불안정하면 `NOWNOTE_BROWSER_PATH`에 Chrome 또는 Edge 실행 파일 경로를 지정합니다.

Windows `.exe` 설치형 프로그램은 `desktop/README.md` 기준으로 생성합니다.
1.1 Windows 설치형은 Web과 같은 화면 계열을 유지하되, Electron 전용 로컬 JSON 저장소와 서버 공유 문서 동기화 흐름을 분리합니다.
설치형 프로그램은 PC 로컬 문서와 서버 공유 문서를 함께 다루는 별도 PC 클라이언트 기준으로 발전시킵니다.

```bash
cd ../desktop
npm install
npm run dist:win
```

생성 결과는 `desktop/dist/NowNote-Setup-0.1.0-x64.exe`입니다.

PWA 설치형 배포 묶음도 개발/검증용으로 아래 명령으로 생성할 수 있습니다.

```bash
python scripts/package_web.py
```

생성 결과는 `web/dist/nownote-web-pwa/`와 `web/dist/nownote-web-pwa.zip`입니다.
배포 묶음은 정적 파일이므로 압축을 푼 뒤 `index.html`을 HTTPS에서 서비스하면 PWA 설치형 점검을 진행할 수 있습니다.

브라우저 실행, PWA 설치, Markdown/JSON 가져오기와 내보내기, 서버 연결까지 포함한 실제 점검은
`runtime_checklist_ko.md`를 기준으로 진행합니다.

서버 제공 Web에서는 로그인 직후 서버의 공유 문서를 먼저 불러오고, 편집 내용은 서버에 다시 반영합니다.
지식/일자별 메모는 개발용 파일 실행 또는 설치형 검증 화면에서 화면 설정의 서버 연결 섹션으로 NowNote 서버와 동기화할 수 있습니다.
서버 연결 테스트 후에는 서버가 지원하는 동기화, 녹음, 분석, 운영 점검, 백업 내보내기, 백업 검증 capability와 공용 서버 준비 상태를 표시합니다.
서버 제공 Web은 사용자 ID와 비밀번호로 `/api/v1/auth/web-login`에 로그인하고, 이후 `X-Now-Web-Session` 세션으로 서버 공유 문서만 접근합니다.
같은 사용자 그룹에 속한 다른 사용자의 공유 지식 메모는 조회 전용으로 함께 표시되며, 제목/본문/속성 수정과 서버 업로드 대상에서 제외됩니다.
신규 사용자는 `/api/v1/auth/register`로 가입하고, 로그인 후 `/api/v1/auth/device-token`에서 앱/설치형 연결 토큰을 발급합니다.
발급한 연결 토큰은 Web 설정 화면에서 다시 확인할 수 있고, 노출됐다고 판단하면 재발급합니다.
비밀번호를 잊은 사용자는 등록 이메일로 `/api/v1/auth/password-reset/request`와 `/api/v1/auth/password-reset/confirm` 흐름을 사용합니다.
설치형 프로그램에서 사용자별 접속 토큰을 입력한 경우 연결 테스트에서 `/api/v1/auth/token-login`까지 확인하며, 2단계 인증 사용자는 6자리 코드를 함께 입력합니다.
브라우저에서 `서버 연결 실패: Failed to fetch`가 보이면 사용자가 입력한 서버 주소가 실제 브라우저에서 열리는 주소인지 확인합니다.
외부 PC와 휴대폰에서는 `localhost`가 아니라 공용 도메인 또는 개인 서버 도메인을 입력해야 합니다.
`서버로 동기화`는 증분 동기화를 수행하고, `전체 다시 동기화`는 마지막 동기화 시점을 초기화해서
서버에 전체 노트를 다시 전송합니다.
1차 설치형 프로그램 포장 방식은 Windows `.exe` 설치 파일과 PWA 설치를 함께 지원하는 방향으로 정리합니다.
`manifest.webmanifest`, `sw.js`, `icons/nownote-icon.svg`를 포함해 브라우저의 설치 기능으로 독립 실행 창을 만들 수 있게 합니다.
서비스 워커는 `https`, `localhost`, `127.0.0.1`에서만 등록되므로 파일을 직접 여는 경우에는 기존 로컬 Web 화면처럼 동작합니다.
Windows `.exe`는 `desktop`의 Electron 패키지로 생성하고, macOS/Linux 설치 파일과 `.msi`는 별도 빌드 도구를 붙이는 단계에서 확장합니다.

서버 연결 후에는 보관된 일자별 메모도 삭제하지 않습니다.
설치형 서버는 사용자의 개인 서버이므로, 보관 메모는 서버에 `archived` 상태의 비활성 백업본으로 동기화합니다.
지식 메모와 일자별 메모에는 서버 동기화를 준비하기 위해 `status`, `syncState`, `updatedAt` 값을 유지합니다.

사진/음성 보관은 NowNote 메모 본체에 넣지 않습니다.
필요해지면 별도 프로그램 또는 별도 모듈로 분리합니다.
