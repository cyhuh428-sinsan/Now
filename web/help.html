로컬 전용

단독 사용자

서버에 연결하지 않고 현재 기기 안에서만 메모를 저장합니다.

  • 바로 시작할 수 있습니다.
  • 메모가 외부 서버로 올라가지 않습니다.
  • JSON 백업이나 Markdown 내보내기를 주기적으로 해두는 것이 좋습니다.
동기화 사용

서버 연결 사용자

개인 Docker 서버나 공용 NowNote 서버에 연결해서 여러 기기와 메모를 동기화합니다.

  • 서버 주소와 API 토큰이 필요합니다.
  • 사용자 프로필, 시간대, 그룹, 활성 상태를 관리할 수 있습니다.
  • 서버 백업, 녹음 저장, 분석 작업으로 확장할 수 있습니다.

플랫폼별 기준

모바일 앱

빠른 입력이 중심입니다. 홈에서 오늘 메모를 빠르게 작성하고, 음성 또는 타이핑으로 기록합니다.

  • 일자별 메모
  • 음성 메모
  • 빠른 메모 입력
  • 서버 연결 상태 확인

Web / 설치형 프로그램

지식 메모가 중심입니다. 주제 / 분류 / 메모 구조로 정리하고 Markdown으로 긴 글을 작성합니다.

  • 계층형 지식 메모
  • 탭 기반 편집
  • 본문 찾기와 빠른 전환
  • 기능별 ON/OFF와 단축키 변경

서버 연결 설정

서버 연결에 필요한 값은 서버 주소, API 토큰, 기기 ID입니다.

서버 주소: http://서버주소:8750
API 토큰: 서버 .env의 NOW_API_TOKEN
기기 ID: 앱 또는 설치형 프로그램에서 자동 생성

Android 에뮬레이터에서 로컬 PC 서버에 연결할 때는 아래 주소를 사용합니다.

http://10.0.2.2:8750

사용자 프로필

서버 연결 사용자는 서버에 사용자 정보가 생성됩니다.

  • owner ID
  • 이메일과 표시 이름
  • 시간대
  • 사용자 그룹
  • 2단계 인증 사용 여부
  • 활성 상태와 최근 접속 시간

앱과 설치형 프로그램은 자기 프로필의 이메일, 표시 이름, 시간대를 수정할 수 있습니다. 그룹, 2단계 인증, 활성 상태는 관리자 화면에서 관리합니다.

백업과 가져오기

JSON 백업

JSON은 NowNote 전체 데이터를 백업하거나 복원할 때 사용합니다. 가져오기는 현재 데이터를 백업 파일 내용으로 교체하는 성격이 강합니다.

Markdown 가져오기

다른 곳에서 작성한 .md, .markdown, .txt 파일을 NowNote 지식 메모로 추가합니다.

가져온 파일은 원본 파일과 연결하지 않습니다. NowNote에서 수정하거나 삭제해도 원본 파일은 그대로 남습니다.

암호화 저장

암호화 저장은 서버 로그인 사용자 전용 선택 기능으로 설계합니다. 기본값은 꺼짐입니다.

  • 로그인은 사용 권한 확인에 사용합니다.
  • 실제 암호화 키는 사용자 비밀번호 또는 복구키 기반으로 분리합니다.
  • 암호화된 메모는 서버 운영자도 원문을 읽을 수 없어야 합니다.
  • 암호화 메모는 기본적으로 LLM 분석에서 제외합니다.

추천 시작 방법

  1. 처음에는 단독 사용자로 메모 구조를 먼저 정합니다.
  2. 일자별 메모와 지식 메모를 사용해 봅니다.
  3. 여러 기기에서 같은 메모가 필요해지면 서버를 연결합니다.
  4. 서버 연결 후 사용자 프로필과 시간대를 확인합니다.
  5. 운영이 안정되면 백업, 분석, 암호화 기능을 단계적으로 켭니다.
