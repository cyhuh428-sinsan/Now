# NowNote Windows 설치형 프로그램

이 폴더는 1차 배포용 Windows `.exe` 설치 파일을 만드는 Electron 패키지입니다.

중요한 기준:

- 화면 디자인은 Web과 같은 계열을 유지합니다.
- 내부 동작은 Web과 달라야 합니다.
- Web은 서버에 부속된 공유 문서 전용 클라이언트입니다.
- 최종 설치형 프로그램은 PC 로컬 문서와 서버 공유 문서를 함께 다루는 별도 PC 클라이언트로 분리합니다.
- 현재 Electron 포장은 1차 Windows 설치형 산출물입니다. 이후 필요하면 로컬 파일 관리와 서버 공유 문서 관리를 더 강하게 분리한 전용 PC 클라이언트로 확장합니다.
- 1.1부터 설치형은 브라우저 localStorage 대신 Electron 전용 로컬 JSON 저장소를 우선 사용합니다.

## 기준

- 1차 Windows 설치형 산출물은 기존 Web 화면을 Electron 앱으로 포장합니다.
- 서버 없이 로컬 단독 사용이 가능합니다.
- 공용 서버 접속값을 입력하면 서버와 동기화할 수 있습니다.
- 설정 화면에서 PC 로컬 저장소 위치를 확인할 수 있습니다.
- 새 지식 메모는 기본적으로 PC 로컬 문서로 저장하고, 사용자가 공유로 전환한 문서만 서버 동기화 대상으로 다룹니다.
- 서버에 이미 공유된 문서를 공유 해제하면 서버에는 삭제 신호를 보내고, PC 로컬 원본은 계속 보존합니다.
- 자동 동기화는 설정 화면에서 켜고 끌 수 있으며, 충돌 문서는 설정 화면 목록에서 로컬 유지/서버 적용/나중에 결정으로 처리합니다.
- Windows 설치 파일은 NSIS 기반 `.exe`로 생성합니다.
- `.msi`는 1차 설치형 범위에 포함하지 않고, WiX 도구를 붙이는 단계에서 추가합니다.

## 개발 실행

```bash
cd desktop
npm install
npm run start
```

`npm run start`는 먼저 `web` 정적 파일을 `desktop/app`으로 복사한 뒤 Electron 창을 엽니다.

## Windows 설치 파일 만들기

```bash
cd desktop
npm install
npm run dist:win
```

생성 위치:

```text
desktop/dist/NowNote-Setup-0.1.0-x64.exe
```

생성된 설치 파일을 실행하면 NowNote가 Windows 시작 메뉴와 바탕화면 바로가기에 등록됩니다.

설치형 로컬 저장소가 재시작 후에도 유지되는지는 설치 파일 생성 후 아래 명령으로 점검합니다.

```bash
cd desktop
npm run check:storage
```

이 점검은 테스트 전용 사용자 데이터 폴더를 만들어 메모를 작성하고, 앱을 재시작한 뒤 같은 메모가 다시 로드되는지 확인합니다.

## 사용자 안내

설치 후 서버 연결 설정을 비워두면 현재 PC에만 저장하는 단독 프로그램으로 사용할 수 있습니다.
로컬 원본 데이터와 화면 설정은 사용자 데이터 폴더의 `nownote-desktop-store.json` 파일에 저장됩니다.
기존에 설치형 포장 화면에서 저장했던 브라우저 localStorage 데이터는 첫 실행 때 파일 저장소로 복사하며, 원본 localStorage 값은 자동 삭제하지 않습니다.
새로 작성한 지식 메모는 기본적으로 PC 안에만 남고, 공유 버튼으로 공유 상태가 된 메모만 서버에 올라갑니다.
삭제 보관함 항목은 로컬 보관함에서 복원하거나 영구 삭제하며, 서버 동기화 대기 개수에는 포함하지 않습니다.
공용 서버에 접속하려면 먼저 Web 프로그램에서 직접 가입하고 로그인한 뒤, 화면 설정의 서버 연결 항목에서 앱/설치형 연결 토큰을 발급합니다.

```text
서버 주소: 예) https://nownote.sinsan.kr
API 토큰: 공용 서버에서는 비움. 개인 서버 기본 모드에서만 서버 관리자 토큰 입력
사용자 ID: Web에서 직접 가입한 사용자 ID
앱/설치형 연결 토큰: Web 로그인 후 직접 발급한 토큰
2단계 인증 코드: 필요한 경우 6자리 코드
```
