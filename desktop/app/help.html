단독 사용자

서버에 연결하지 않고 현재 기기 안에서만 메모를 저장합니다.

  • 바로 시작할 수 있습니다.
  • 메모가 외부 서버로 올라가지 않습니다.
  • 설치형/개발용 Web은 JSON 백업, 모바일은 DB 백업을 주기적으로 해두는 것이 좋습니다.
동기화 사용

서버 연결 사용자

개인 Docker 서버나 공용 NowNote 서버에 연결해서 여러 기기와 메모를 동기화합니다.

  • 앱/설치형은 Web에서 발급한 연결 토큰을 쓰고, Web은 사용자 ID와 비밀번호로 로그인합니다.
  • 사용자 프로필, 시간대, 그룹, 활성 상태를 관리할 수 있습니다.
  • 서버 백업, 녹음 저장, 분석 작업으로 확장할 수 있습니다.
  • 서버 백업 검증으로 스키마, 체크섬, 필수 항목, 토큰 민감정보 노출 여부를 확인합니다.
  • 공용 서버용 사용자 직접 가입, Web 로그인, 연결 토큰 발급, 비밀번호 재설정, 2단계 코드 검증을 제공합니다.
  • 공용 서버 운영 전에는 Web 직접 가입, 앱/설치형 연결 토큰 발급, 공개 HTTPS, reverse proxy 환경을 확인해야 합니다.

플랫폼별 기준

모바일 앱

빠른 입력이 중심입니다. 홈에서 오늘 메모를 빠르게 작성하고, 음성 또는 타이핑으로 기록합니다.

  • 일자별 메모
  • 음성 메모
  • 빠른 메모 입력
  • 서버 연결 상태 확인
  • 서버 녹음 업로드 상태 확인
  • 서버 분석 작업 등록과 결과 확인

Web / 설치형 프로그램

디자인은 비슷하지만 내부 동작은 다릅니다. Web은 서버 공유 문서만 접속하고, 설치형 프로그램은 PC 로컬 문서와 공유 문서를 함께 다룹니다.

설치형 프로그램은 PC 로컬 저장소를 우선 사용하고, 공유로 선택한 지식 메모만 서버와 동기화합니다.

  • 계층형 지식 메모
  • 탭 기반 편집
  • 본문 찾기와 빠른 전환
  • 전체 그래프, 로컬 그래프, 고립/허브 메모, 연결 후보
  • 기능별 ON/OFF와 단축키 변경
  • 서버 분석 결과 확인과 메모 본문 추가

서버 연결 설정

설치형 프로그램과 모바일 앱은 서버 주소, 사용자 ID, 앱/설치형 접속 토큰으로 서버와 동기화합니다. Web은 서버 주소에서 사용자 ID와 비밀번호로 로그인합니다.

서버를 설치하지 않고 단독으로 쓰는 경우 모바일 앱과 설치형 프로그램의 서버 연결 설정은 비워둡니다. Web은 서버에 부속된 화면이므로 단독 로컬 저장용으로 쓰지 않습니다.
공용 서버에 접속만 하는 사용자는 서버를 직접 설치하지 않습니다. Web에서 직접 가입하고, 설치형 프로그램/모바일 앱은 Web에서 발급한 연결 토큰을 입력합니다.
Web: 사용자 ID + 비밀번호
설치형/앱 서버 주소: http://서버주소:8750 또는 https://도메인
설치형/앱 접속 토큰: Web 로그인 후 화면 설정에서 발급/재확인
2단계 인증 코드: 연결 확인 때만 필요한 경우 6자리 코드
구형 개인 서버 API 토큰: 기존 개인 서버 호환이 필요할 때만 고급 설정에서 사용

Android 에뮬레이터에서 로컬 PC 서버에 연결할 때는 아래 주소를 사용합니다.

http://10.0.2.2:8750

사용자 프로필

서버 연결 사용자는 서버에 사용자 정보가 생성됩니다.

  • owner ID
  • 이메일과 표시 이름
  • 시간대
  • 사용자 그룹
  • 2단계 인증 사용 여부
  • 활성 상태와 최근 접속 시간

앱과 설치형 프로그램은 자기 프로필의 이메일, 표시 이름, 시간대를 수정할 수 있습니다. Web 사용자는 그룹 목록을 확인하고 관리자에게 받은 초대코드로 그룹에 참가할 수 있으며, 2단계 인증과 활성 상태는 관리자 화면에서 관리합니다.

2단계 인증을 사용하는 사용자는 토큰 로그인 확인 때 6자리 인증 코드를 함께 입력해야 합니다.

백업과 가져오기

설치형 / 개발용 Web JSON 백업

JSON은 설치형 또는 개발용 Web 데이터를 백업하거나 복원할 때 사용합니다. 공용 Web 프로그램의 원본은 서버 공유 문서이므로 서버 백업 절차를 우선합니다.

모바일 DB 백업

모바일 앱은 .db 파일로 전체 데이터를 백업하거나 복원합니다. 가져오기는 현재 모바일 데이터를 백업 파일 내용으로 교체하며, 완료 후 앱을 다시 실행해야 합니다.

Markdown 가져오기

다른 곳에서 작성한 .md, .markdown, .txt 파일을 NowNote 지식 메모로 추가합니다.

가져온 파일은 원본 파일과 연결하지 않습니다. NowNote에서 수정하거나 삭제해도 원본 파일은 그대로 남습니다.

Markdown 내보내기

NowNote에서 내보낸 Markdown은 현재 언어에 맞춰 생성됩니다. 한국어 내보내기와 영어 내보내기 모두 다시 가져올 수 있습니다.

암호화 저장

지식 메모는 필요할 때 메모 단위로 암호화할 수 있습니다. 같은 메모를 Web, 설치형 프로그램, 앱에서 열려면 같은 키를 입력해야 합니다.

  • 로그인은 사용 권한 확인에 사용합니다.
  • 암호화 키는 브라우저, 앱, 서버에 저장하지 않으므로 키를 잊으면 원문을 복구할 수 없습니다.
  • 암호화된 메모는 서버 운영자도 원문을 읽을 수 없어야 합니다.
  • 암호화 메모는 기본적으로 LLM 분석에서 제외합니다.

추천 시작 방법

  1. 처음에는 단독 사용자로 메모 구조를 먼저 정합니다.
  2. 일자별 메모와 지식 메모를 사용해 봅니다.
  3. 여러 기기에서 같은 메모가 필요해지면 서버를 연결합니다.
  4. 서버 연결 후 사용자 프로필과 시간대를 확인합니다.
  5. 운영이 안정되면 백업, 분석, 암호화 기능을 단계적으로 켭니다.
