# NowNote 모바일 실제 실행 점검서

이 문서는 Android 에뮬레이터와 실제 Android 기기에서 NowNote 1차 모바일 흐름을 확인하기 위한 점검서입니다.
자동 정적 점검은 `scripts/verify_mobile_surface.py`가 담당하고, 이 문서는 사람이 화면에서 확인해야 하는 항목을 다룹니다.

## 점검 전 준비

- 최신 소스 반영: 저장소 루트에서 `git pull origin main`
- Flutter 의존성 준비: `now_app` 폴더에서 `flutter pub get`
- 실제 실행 환경 확인: `now_app` 폴더에서 `python scripts/check_android_runtime.py`
- 에뮬레이터 준비 확인: `now_app` 폴더에서 `python scripts/check_android_emulator.py`
- 에뮬레이터 자동 시작과 앱 실행 확인: `now_app` 폴더에서 `python scripts/check_android_emulator.py --start --launch-app`
- 이미 설치된 앱 실행 확인: `now_app` 폴더에서 `python scripts/check_android_emulator.py --launch-app --skip-install`
- APK 설치와 앱 실행 자동 확인: `now_app` 폴더에서 `python scripts/check_android_launch.py`
- `Flutter CLI`가 버전은 출력하지만 종료가 늦어 `WARN`으로 표시되는 경우, Flutter 설치 자체는 확인된 상태로 보고 연결 기기와 서버 상태를 우선 점검한다.
- 로컬 단독 사용 점검: 서버 연결을 끈 상태로 시작
- 서버 연결 점검: 개인 Docker 서버 또는 테스트 서버를 먼저 실행
- Android 에뮬레이터에서 Windows/WSL 서버에 연결할 때 서버 주소는 보통 `http://10.0.2.2:8750`
- 실제 Android 기기에서는 같은 네트워크의 PC IP 또는 실제 서버 도메인 사용

## 1. 앱 실행과 홈

- [ ] `python scripts/check_android_emulator.py`가 ADB, emulator CLI, AVD 목록을 확인한다.
- [ ] `python scripts/check_android_launch.py`가 연결된 에뮬레이터 또는 실기기에서 통과한다.
- [ ] 설치가 필요 없는 재확인은 `python scripts/check_android_emulator.py --launch-app --skip-install`로 통과한다.
- [ ] 앱 이름이 NowNote로 표시된다.
- [ ] 홈 화면에 오늘 일정과 오늘 메모 영역이 보인다.
- [ ] 홈의 오늘 메모 진입 버튼으로 일자별 메모 작성 화면에 들어간다.
- [ ] 하단 탭의 기존 살림, 여행, 회의, 대화 흐름이 유지된다.

## 2. 일자별 메모

- [ ] 오늘 날짜의 메모장을 연다.
- [ ] 텍스트를 입력하고 저장한다.
- [ ] 다시 들어갔을 때 같은 날짜의 메모가 유지된다.
- [ ] 같은 날짜에 추가 입력하면 새 메모가 아니라 같은 메모장에 이어서 저장된다.
- [ ] 다른 날짜를 선택하면 해당 날짜 메모를 따로 볼 수 있다.

## 3. 계층 메모

- [ ] 주제를 추가한다.
- [ ] 주제 아래 분류를 추가한다.
- [ ] 분류 아래 메모를 추가한다.
- [ ] 3단계 메모 아래에는 더 이상 하위 메모를 추가할 수 없다.
- [ ] 하위 메모가 있는 상위 메모는 바로 삭제되지 않는다.
- [ ] 하위 메모가 없는 메모는 삭제 보관함으로 이동한다.
- [ ] 삭제 보관함에서 항목을 확인한다.

## 4. 음성 메모

- [ ] 일자별 메모에서 실시간 변환으로 음성을 입력한다.
- [ ] 일자별 메모에서 녹음 후 변환으로 음성을 입력한다.
- [ ] 계층 메모에서 실시간 변환으로 음성을 입력한다.
- [ ] 계층 메모에서 녹음 후 변환으로 음성을 입력한다.
- [ ] 녹음 후 변환 모드에서는 녹음 파일과 텍스트 결과가 함께 남는지 확인한다.

## 5. Markdown 가져오기

- [ ] `.md`, `.markdown`, `.txt` 파일을 선택한다.
- [ ] 가져온 파일이 새 지식 메모로 추가된다.
- [ ] 가져온 뒤 원본 파일을 수정하지 않아도 NowNote 내부 메모가 유지된다.
- [ ] NowNote에서 해당 메모를 수정하거나 삭제해도 원본 파일은 바뀌지 않는다.

## 6. 서버 연결

- [ ] 서버 설정에서 단독 사용과 서버 연결 모드를 전환할 수 있다.
- [ ] 서버 주소, API 토큰, 사용자별 접속 토큰, 사용자 ID, 기기 ID를 입력할 수 있다.
- [ ] 2단계 인증 코드는 저장되지 않고 연결 테스트 요청에만 사용된다.
- [ ] 연결 테스트가 성공한다.
- [ ] 메모 동기화를 실행한다.
- [ ] 서버의 `/admin` 또는 `/monitor` 화면에서 동기화된 메모를 확인한다.
- [ ] 녹음이 있는 경우 녹음 업로드 상태를 확인한다.

## 7. 실패 시 확인 기준

- 서버 연결 실패가 나오면 에뮬레이터에서는 `localhost` 대신 `10.0.2.2`를 사용했는지 확인한다.
- 실제 기기에서는 휴대폰과 서버 PC가 같은 네트워크에 있는지 확인한다.
- 공용 서버는 HTTPS와 reverse proxy 설정이 끝난 뒤 점검한다.
- 사용자별 접속 토큰 필수 서버에서는 사용자 토큰이 없거나 비활성 사용자면 실패가 정상이다.
- 2단계 인증 사용자는 현재 시각 기준의 6자리 코드를 입력해야 한다.

## 완료 기록

점검이 끝나면 `docs/PHASE1_RELEASE_CHECKLIST.md`의 모바일 앱 실제 점검 항목을 실제 결과에 맞춰 갱신합니다.
